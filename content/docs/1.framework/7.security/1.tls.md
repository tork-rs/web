# TLS and HTTP/2

Tork can terminate TLS itself and tune the HTTP/1 and HTTP/2 connection layers. TLS lives behind the `tls` feature, which is off by default so the base build stays lean.

```toml
tork = { version = "0.1", features = ["tls"] }
```

It uses rustls with the ring provider, so there is no OpenSSL and no C toolchain.

## Enabling TLS

Pass a `TlsConfig` to `App::tls`. Build the config from PEM file paths with `from_pem_files`, or from PEM contents with `from_pem`.

```rust
use tork::{App, TlsConfig};

App::new()
    .include(index)
    .tls(TlsConfig::from_pem_files("./cert.pem", "./key.pem")?)
    .serve("0.0.0.0:8443")
    .await?;
```

`cert.pem` is a certificate chain (leaf first), and `key.pem` is a PKCS#8, PKCS#1, or SEC1 private key. A malformed certificate or key fails fast at boot, not on the first connection, and the startup log then shows `https://`.

HTTP/2 is negotiated automatically: ALPN advertises `h2` then `http/1.1`, so a modern client gets HTTP/2 and older clients fall back to HTTP/1.1. Call `.http1_only()` on the config to advertise only HTTP/1.1, or `.alpn(protocols)` to set the list yourself.

## Trying it locally

Generate a self-signed certificate for `localhost`:

```bash
openssl req -x509 -newkey rsa:2048 -nodes -days 365 \
  -keyout key.pem -out cert.pem -subj "/CN=localhost"
```

Run the app, then call it (with `-k`, since the certificate is self-signed):

```bash
# Confirm the connection is HTTP/2 over TLS.
curl -k --http2 -I https://localhost:8443/
```
