# Authentication

Authentication answers one question: who is this request from. The usual shape for an API is that a client sends a username and password once, gets back a signed token, and then sends that token on every later request. Tork gives you the pieces to read and route on that token. You sign and verify it with standard crates, the same way FastAPI leaves that to PyJWT and pwdlib.

This guide builds the whole flow: a `POST /token` login endpoint that returns a JWT, and protected endpoints that read it through the `BearerToken` extractor and a dependency guard. The complete, compiling code is the `auth_jwt` example in the framework repository.

## What the framework gives you, and what you add

Tork provides the request-side pieces:

- `BearerToken`, an extractor that reads the `Authorization: Bearer <token>` header and answers `401 Unauthorized` when it is missing or malformed.
- `#[tork::dependency]`, which turns a "current user" type into something handlers ask for by type.
- `Error::unauthorized` (401) and `Error::forbidden` (403), and `#[settings]` plus `SecretString` for the signing key.

You add two ordinary crates, because the framework stays out of cryptography:

- `jsonwebtoken` to sign and verify JWTs.
- `argon2` to hash and check passwords.

```toml
[dependencies]
tork = "0.1"
jsonwebtoken = "9"
argon2 = { version = "0.5", features = ["std"] }
serde = { version = "1", features = ["derive"] }
```

## Configure the signing secret

A JWT is signed with a secret key, and the same key verifies it later. Keep it out of the code with `#[settings]` and `SecretString`, which masks the value in logs and never serializes it.

```rust
use tork::{settings, SecretString};

#[settings(prefix = "AUTH")]
pub struct AuthConfig {
    /// The HMAC key used to sign and verify JWTs.
    #[setting(secret, default = "dev-only-change-me")]
    pub jwt_secret: SecretString,
    /// How long an access token stays valid, in minutes.
    #[setting(default = 30, ge = 1, le = 1440)]
    pub access_token_ttl_minutes: u64,
}
```

Load it once at startup and register it so handlers and guards receive it as `Arc<AuthConfig>`:

```rust
let config = AuthConfig::load()?;
App::new().state(std::sync::Arc::new(config));
```

Set `AUTH_JWT_SECRET` in production; the default exists only so the example runs without setup.

## Hash passwords

Never store a plaintext password. Hash it with Argon2 when a user is created, and compare hashes on login. The hash carries its own random salt, so two users with the same password get different hashes.

```rust
use argon2::password_hash::rand_core::OsRng;
use argon2::password_hash::{PasswordHash, PasswordHasher, PasswordVerifier, SaltString};
use argon2::Argon2;
use tork::Error;

pub fn hash_password(password: &str) -> tork::Result<String> {
    let salt = SaltString::generate(&mut OsRng);
    Argon2::default()
        .hash_password(password.as_bytes(), &salt)
        .map(|hash| hash.to_string())
        .map_err(|error| Error::internal(format!("failed to hash password: {error}")))
}

pub fn verify_password(password: &str, hash: &str) -> bool {
    match PasswordHash::new(hash) {
        Ok(parsed) => Argon2::default()
            .verify_password(password.as_bytes(), &parsed)
            .is_ok(),
        Err(_) => false,
    }
}
```

One note on timing: checking the password only when the user exists leaks, through response time, whether a username is registered. A hardened login also hashes a throwaway password when the user is missing, so both paths take a similar amount of time. The example keeps the simple version; add that step for production.

## Sign and verify JWTs

A JWT carries a small JSON payload, the claims, signed with the secret. Here the claims are the user id (`sub`), an expiry timestamp (`exp`), and the scopes the token grants.

```rust
use jsonwebtoken::{decode, encode, Algorithm, DecodingKey, EncodingKey, Header, Validation};
use serde::{Deserialize, Serialize};
use std::time::{SystemTime, UNIX_EPOCH};

#[derive(Debug, Serialize, Deserialize)]
pub struct Claims {
    pub sub: String,
    pub exp: usize,
    pub scopes: Vec<String>,
}

pub fn encode_token(config: &AuthConfig, user_id: i64, scopes: Vec<String>) -> tork::Result<String> {
    let now = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .map(|d| d.as_secs())
        .unwrap_or(0);
    let exp = now + config.access_token_ttl_minutes * 60;

    let claims = Claims { sub: user_id.to_string(), exp: exp as usize, scopes };

    encode(
        &Header::new(Algorithm::HS256),
        &claims,
        &EncodingKey::from_secret(config.jwt_secret.expose().as_bytes()),
    )
    .map_err(|error| tork::Error::internal(format!("failed to sign token: {error}")))
}

pub fn decode_token(config: &AuthConfig, token: &str) -> tork::Result<Claims> {
    decode::<Claims>(
        token,
        &DecodingKey::from_secret(config.jwt_secret.expose().as_bytes()),
        &Validation::new(Algorithm::HS256),
    )
    .map(|data| data.claims)
    .map_err(|_| tork::Error::unauthorized("invalid or expired token"))
}
```

`jsonwebtoken` verifies the signature and the `exp` claim for you, so a bad or expired token becomes a `401`.

## The login endpoint

`POST /token` checks the password and returns a signed token. The request body is a validated `#[api_model]`.

```rust
use tork::api_model;

#[api_model]
pub struct LoginInput {
    #[field(min_length = 1, max_length = 50)]
    pub username: String,
    #[field(min_length = 1, max_length = 200)]
    pub password: String,
}

#[api_model]
pub struct TokenResponse {
    pub access_token: String,
    pub token_type: String,
}
```

```rust
use std::sync::Arc;
use tork::{post, Error, Valid};

#[post("/token", response_model = TokenResponse, summary = "Log in and get a token")]
pub async fn login(
    repo: UserRepository,
    config: Arc<AuthConfig>,
    body: Valid<LoginInput>,
) -> tork::Result<TokenResponse> {
    let input = body.into_inner();

    let user = repo
        .find_by_username(&input.username)
        .filter(|user| verify_password(&input.password, &user.password_hash))
        .ok_or_else(|| Error::unauthorized("incorrect username or password"))?;

    let token = encode_token(&config, user.id, user.scopes.clone())?;
    Ok(TokenResponse { access_token: token, token_type: "bearer".to_owned() })
}
```

`UserRepository` is a normal `#[derive(Inject)]` repository over your user store (see [Repository pattern](/docs/orm/advanced/repository-pattern)). The same response works whether your users live in memory, in the ORM, or anywhere else.

## Read the token: the current-user guard

A protected handler should receive the authenticated user, not a raw token. Build a `CurrentUser` with `#[tork::dependency]`. It depends on `BearerToken`, the repository, and the config; Tork resolves those first, then runs `resolve`, which decodes the token, loads the user, and rejects a disabled account.

```rust
use std::sync::Arc;
use tork::{BearerToken, Error};

#[derive(Clone)]
pub struct CurrentUser {
    pub id: i64,
    pub username: String,
    pub role: String,
    pub scopes: Vec<String>,
}

#[tork::dependency]
impl CurrentUser {
    pub async fn resolve(
        token: BearerToken,
        repo: UserRepository,
        config: Arc<AuthConfig>,
    ) -> tork::Result<Self> {
        let claims = decode_token(&config, token.token())?;
        let user_id: i64 = claims
            .sub
            .parse()
            .map_err(|_| Error::unauthorized("malformed token subject"))?;
        let user = repo.find_by_id(user_id)?;

        if user.disabled {
            return Err(Error::forbidden("this account is disabled"));
        }

        Ok(CurrentUser {
            id: user.id,
            username: user.username,
            role: user.role,
            scopes: claims.scopes,
        })
    }
}
```

Now any handler that takes a `CurrentUser` parameter is protected: a missing or invalid token rejects the request before the handler body runs, so inside the handler the user is always present and valid.

```rust
use tork::get;

#[get("/users/me", response_model = UserOut, security = ["bearerAuth"], summary = "Current user")]
pub async fn read_current_user(current_user: CurrentUser) -> tork::Result<UserOut> {
    Ok(UserOut {
        id: current_user.id,
        username: current_user.username,
        role: current_user.role,
    })
}
```

## The Authorize button in the docs

Declare the bearer scheme on the OpenAPI document with `bearer_auth()`, and mark protected routes with `security = ["bearerAuth"]` (as on the handler above). The documentation UI at `/docs` then shows an "Authorize" button where a reader pastes a token, and a lock on each protected operation.

```rust
use tork::{App, OpenApi};

App::new()
    .state(std::sync::Arc::new(config))
    .state(std::sync::Arc::new(user_store))
    .include_router(routers::router())
    .openapi(
        OpenApi::new()
            .title("Auth Example")
            .version("1.0.0")
            .bearer_auth()              // registers the scheme -> Authorize button
            .json("/openapi.json")
            .docs("/docs"),
    );
```

`bearer_auth()` adds `{ "type": "http", "scheme": "bearer", "bearerFormat": "JWT" }` to the document, and each `security = ["bearerAuth"]` route records that it requires the scheme.

## Try it

Run the example, log in (the seeded user is `ada` / `secret`), and call a protected route with the token:

```bash
# 1. Log in. The response is { "access_token": "...", "token_type": "bearer" }.
curl -s -X POST http://localhost:8000/token \
  -H 'content-type: application/json' \
  -d '{"username":"ada","password":"secret"}'

# 2. Copy the access_token, then call a protected route with it.
curl -s http://localhost:8000/users/me \
  -H 'Authorization: Bearer <paste-token-here>'
```

Or open `http://localhost:8000/docs`, click Authorize, paste the token, and call `GET /users/me`.

## Test it

The flow runs in process with `TestClient`, no server needed:

```rust
#[tokio::test]
async fn login_then_read_current_user() -> tork::Result<()> {
    let client = client().await; // App::new().state(config).state(store).include_router(...)

    let res = client
        .post("/token")
        .json(&serde_json::json!({ "username": "ada", "password": "secret" }))
        .send()
        .await?;
    assert_eq!(res.status(), 200);
    let token = res.json::<serde_json::Value>().await?["access_token"]
        .as_str()
        .unwrap()
        .to_owned();

    let me = client
        .get("/users/me")
        .header("authorization", &format!("Bearer {token}"))
        .send()
        .await?;
    assert_eq!(me.status(), 200);

    // No token at all is rejected with 401.
    assert_eq!(client.get("/users/me").send().await?.status(), 401);
    Ok(())
}
```
