# Ask for what you need in the signature

Each extractor validates its slice of the request before your function body
runs. Compose as many as you need — the handler only ever sees well-formed data.

```rust
#[post("/users/:id")]
async fn update(
    Path(id): Path<u64>,
    Query(opts): Query<Opts>,
    Json(body): Json<UpdateUser>,
) -> Json<User> {
    Json(db::update(id, opts, body).await)
}
```

## Built-in extractors

| Extractor | Source | Failure |
| --- | --- | --- |
| `Path<T>` | Typed URL segments | `404 Not Found` |
| `Query<T>` | Deserialized query string | `400 Bad Request` |
| `Json<T>` | Request body, checked against your struct | `400 Bad Request` |
| `State<T>` | Shared application state | infallible |

## How it works

An extractor is any type that implements `FromRequest`. Tork runs each one in
order; the first that fails short-circuits with its status code, so your body is
reached only when every argument is valid.

Write your own by implementing `FromRequest` for a type — the same mechanism the
built-ins use.
