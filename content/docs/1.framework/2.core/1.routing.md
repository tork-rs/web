# Routes are plain async functions

The `#[get]` attribute binds a path and method to a handler. The return type
decides serialization; the argument list decides what gets pulled from the
request. Register handlers on a `Router` and the table is built at compile time.

```rust
#[get("/users/:id")]
async fn show(Path(id): Path<u64>) -> Json<User> {
    // `id` is already parsed to u64 — a bad path 404s before we run.
    Json(db::find(id).await)
}

let app = Router::new()
    .route(show)
    .nest("/v1", api())
    .layer(Trace::new());
```

## Methods

Every HTTP method has a matching attribute: `#[get]`, `#[post]`, `#[put]`,
`#[patch]`, and `#[delete]`. The path may contain typed segments (`:id`) and a
trailing wildcard (`*rest`).

## Nesting and layers

- `nest(prefix, router)` mounts a sub-router under a path prefix.
- `layer(middleware)` wraps every route below it — see
  [middleware](/docs/framework/core/middleware).

Because the table is resolved when you compile, overlapping or unreachable
routes are reported as build errors rather than surprising you at runtime.
