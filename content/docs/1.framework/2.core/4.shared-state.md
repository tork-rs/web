# Shared state

Most services need a database pool, a config, or a cache available to every
handler. Register it once on the router and pull it out with `State<T>`.

```rust
#[derive(Clone)]
struct AppState {
    db: Pool,
}

#[get("/users")]
async fn list(State(state): State<AppState>) -> Json<Vec<User>> {
    Json(User::query().all(&state.db).await)
}

#[tokio::main]
async fn main() {
    let state = AppState { db: Pool::connect("postgres://…").await };
    Router::new()
        .route(list)
        .with_state(state)
        .listen("0.0.0.0:8080").await;
}
```

## Models and the pool

A `Model`-derived struct queries through the pool you stored in state. The
columns are checked at compile time against the struct that defined the table:

```rust
let active = User::query()
    .filter(User::is_active.eq(true))
    .order_by(User::username)
    .all(&state.db).await?; // Vec<User>
```

State is cloned per request, so keep it cheap — wrap large or shared resources
in an `Arc`. Anything missing at registration is a compile error, not a panic.
