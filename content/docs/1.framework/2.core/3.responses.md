# The return type is the contract

A handler's return type decides how the response is shaped. Return a string for
plain text, `Json<T>` for a serialized body, or a tuple to set the status code.

```rust
use tork::{get, Json, StatusCode};

#[get("/teapot")]
async fn teapot() -> (StatusCode, Json<Message>) {
    (StatusCode::IM_A_TEAPOT, Json(Message { text: "short and stout" }))
}
```

## Results map to status codes

Return a `Result` and Tork turns the `Err` into a response through your error
type's `IntoResponse`:

```rust
#[get("/users/:id")]
async fn show(Path(id): Path<u64>) -> Result<Json<User>, ApiError> {
    let user = db::find(id).await.ok_or(ApiError::NotFound)?;
    Ok(Json(user))
}
```

## Custom responses

Implement `IntoResponse` for any type to control status, headers, and body in
one place. Because the conversion is type-directed, a handler can never return a
shape the client wasn't promised.
