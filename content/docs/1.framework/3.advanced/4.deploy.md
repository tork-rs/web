# Deploy

A Tork service compiles to one binary with no runtime dependencies, so a
production image can be tiny. Build static, copy into `scratch`, and run.

```dockerfile
FROM rust:1.75 AS build
WORKDIR /app
COPY . .
RUN cargo build --release

FROM scratch
COPY --from=build /app/target/release/myproject /myproject
EXPOSE 8080
ENTRYPOINT ["/myproject"]
```

## Migrations on release

Apply migrations as a release step before the new version takes traffic. State
lives in `_tork_migrations`, so re-running is safe and `status` never lies:

```bash
tork-orm migrate up
# → 0001_add_users  applying…
# ✓ 0001_add_users  migrated (12ms)
```

## Graceful shutdown

The server listens for `SIGTERM` and drains in-flight requests before exiting,
so rolling deploys finish cleanly:

```rust
Router::new()
    .route(index)
    .listen("0.0.0.0:8080")
    .with_graceful_shutdown(shutdown_signal())
    .await;
```

That is the whole lifecycle — build once, migrate on release, serve, and drain
on exit.
