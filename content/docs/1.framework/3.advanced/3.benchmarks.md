# Datasheet

Measured on a single 4-core instance, release build, keep-alive connections,
JSON echo workload. Numbers are medians over 60 seconds.

## Headline figures

| Metric | Value |
| --- | --- |
| Throughput | 142k req / s |
| Latency p50 | 0.21 ms |
| Latency p99 | 1.9 ms |

## Material datasheet

| Spec | Value |
| --- | --- |
| Idle memory | 4.8 MB |
| Cold start | 11 ms |
| Release binary | 3.1 MB |
| Runtime | async / tokio |
| Unsafe blocks in core | 0 |

## Reproducing

The workload is an echo handler under `wrk` with keep-alive. Build in release
and pin the server to the cores you measure:

```bash
cargo build --release
taskset -c 0-3 ./target/release/echo &
wrk -t4 -c256 -d60s --latency http://127.0.0.1:8080/echo
```

Numbers vary with hardware and kernel tuning; treat them as a relative baseline,
not an absolute guarantee.
