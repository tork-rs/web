# WebSockets

A WebSocket route looks like any other handler — it takes a `WebSocket`
extractor and returns the upgraded connection. Tork handles the handshake.

```rust
use tork::{get, ws::{WebSocket, Message}};

#[get("/ws")]
async fn echo(mut socket: WebSocket) {
    while let Some(Ok(msg)) = socket.recv().await {
        if let Message::Text(text) = msg {
            let _ = socket.send(Message::Text(text)).await;
        }
    }
}
```

## Splitting read and write

For full-duplex work, split the socket into a sink and a stream and drive them
from separate tasks:

```rust
let (mut tx, mut rx) = socket.split();
tokio::spawn(async move {
    while let Some(Ok(msg)) = rx.next().await {
        // handle inbound
    }
});
tx.send(Message::Text("ready".into())).await?;
```

Connections are backed by the same Tokio runtime as the rest of your service,
so they share its scheduler and shutdown signal.
