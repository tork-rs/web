# Streaming responses

Return a `Stream` and Tork writes the body incrementally, without buffering the
whole payload in memory. This suits large exports and server-sent events.

```rust
use tork::{get, response::Sse, sse::Event};
use futures::stream::{self, StreamExt};

#[get("/events")]
async fn events() -> Sse<impl Stream<Item = Event>> {
    let stream = stream::repeat_with(|| Event::data("tick"))
        .throttle(Duration::from_secs(1));
    Sse::new(stream)
}
```

## Chunked bodies

For a plain chunked response, return a `Body` built from any byte stream:

```rust
#[get("/export.csv")]
async fn export() -> Body {
    Body::from_stream(db::rows().map(|r| r.to_csv_line()))
}
```

Backpressure is preserved end to end: if the client reads slowly, the stream is
polled slowly, and memory stays flat regardless of payload size.
