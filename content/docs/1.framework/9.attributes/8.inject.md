# `#[derive(Inject)]`, `#[inject(...)]`, and `#[logger(...)]`

Deriving `Inject` makes a struct buildable from the request: each field is resolved as a
dependency, then the struct is assembled and passed to your handler. Use it for service
objects that hold resources and other services.

**Applies to:** a struct with named fields.

```rust
use tork::Inject;

#[derive(Inject)]
pub struct UserService {
    repo: UserRepository,
}

impl UserService {
    pub async fn get(&self, id: i64) -> tork::Result<UserOut> {
        self.repo.find(id).await
    }
}
```

## How fields are resolved

Every field is resolved through `FromRequest`, so a field may be a resource, another
`Inject` service, a built-in extractor, or a `#[tork::dependency]` type. Tork resolves
the whole graph and hands you the finished service.

## Logger context

A field of type `Logger` is resolved and then bound to a context string, which appears in
its log lines. The context is chosen in this order:

| Source | Example |
| --- | --- |
| A field-level `#[logger(context = "...")]` | `#[logger(context = "users")] log: Logger` |
| A struct-level `#[inject(context = "...")]` | `#[inject(context = "service")]` on the struct |
| The struct's name, if neither is given | `UserService` |

`context` is the only key accepted by `#[inject]` and `#[logger]`.

```rust
use tork::{Inject, Logger};

#[derive(Inject)]
#[inject(context = "users")]
pub struct UserService {
    repo: UserRepository,

    #[logger(context = "users.audit")]
    log: Logger,
}
```
