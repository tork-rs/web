# `#[middleware]`

Turns an `async fn` into a middleware layer: code that runs around request handling and
can inspect or change the request and the response. The macro generates a type you pass to
`App::middleware`.

**Applies to:** an `async fn` that takes exactly two parameters, `(request, next)`, and
returns `tork::Result<Response>`.

**Options:** none.

```rust
use tork::{middleware, Next, Request, Response, Result};

#[middleware]
async fn add_process_time(req: Request, next: Next) -> Result<Response> {
    let start = std::time::Instant::now();
    let mut response = next.run(req).await?;
    let elapsed = start.elapsed().as_secs_f64();
    if let Ok(value) = tork::HeaderValue::from_str(&format!("{elapsed:.6}")) {
        response.headers_mut().insert(
            tork::HeaderName::from_static("x-process-time"),
            value,
        );
    }
    Ok(response)
}

// usage: App::new().middleware(add_process_time)
```

## Rules

- The function must be `async`, cannot take `self`, and must take exactly two parameters.
- Call `next.run(request).await` to continue down the chain. Returning early without
  calling it short-circuits the request.
- The generated value is a unit type with the same name, so you register it by name with
  `App::middleware(...)`.
