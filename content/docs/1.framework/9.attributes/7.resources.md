# `#[derive(Resources)]` and `#[resource]`

A resources struct holds the long-lived values your application shares: a database pool,
an in-memory store, a broadcast channel. Deriving `Resources` and marking fields with
`#[resource]` registers each marked field by its type, so handlers and services can ask
for it directly. This is the container that a [lifespan](/docs/framework/attributes/lifespan)
builds.

**Applies to:** a struct with named fields. A unit struct (with no fields) is allowed and
registers nothing. Tuple structs and enums are rejected.

**Options:** `#[resource]` is a plain marker; it takes no value. Fields without it are
ignored.

```rust
use tork::Resources;

#[derive(Clone, Resources)]
pub struct AppState {
    #[resource]
    pub store: UserStore,

    #[resource]
    pub config: std::sync::Arc<Config>,

    // Not marked, so not registered.
    pub label: String,
}
```

## What it generates

- A registration step that inserts a clone of each `#[resource]` field by its type.
- A `FromRequest` implementation for each `#[resource]` field type, so it can be injected
  directly (`store: UserStore`) or used as a field of an [`Inject`](/docs/framework/attributes/inject)
  service.
- A field typed `Arc<T>` is registered but does not get its own `FromRequest`; it is
  injected through the built-in `FromRequest for Arc<T>` instead. This is the cheap way to
  share an immutable value such as configuration.
