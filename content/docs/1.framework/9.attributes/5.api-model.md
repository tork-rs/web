# `#[api_model]` and `#[field(...)]`

`#[api_model]` turns a struct into a request or response model. It derives `Debug`,
`Clone`, serialization (`serde`), validation (`garde`), and a JSON Schema (`schemars`)
in one place, so the same type can be read from a request body, returned as a response,
and shown in the OpenAPI document.

**Applies to:** a struct with named fields. Generics are supported.

## `#[api_model]` options

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| `rename_all` | a string such as `"camelCase"` | none | Renames every field in the serialized JSON and the schema. The value is passed to `serde`, so any casing `serde` supports is allowed. |

Unknown options are a compile error.

```rust
use tork::api_model;

#[api_model(rename_all = "camelCase")]
pub struct UserOut {
    pub id: i64,
    pub display_name: String,
}
```

## `#[field(...)]` rules

`#[field(...)]` attaches validation rules and schema metadata to a single field. A field
with no `#[field]` simply has no rules.

| Rule | Value | Effect |
| --- | --- | --- |
| `min_length` | an integer | Minimum string length. |
| `max_length` | an integer | Maximum string length. |
| `ge` | a number | Value must be greater than or equal to this. |
| `le` | a number | Value must be less than or equal to this. |
| `gt` | a number | Value must be strictly greater than this. |
| `lt` | a number | Value must be strictly less than this. |
| `title` | a string | Schema title (documentation only). |
| `description` | a string | Schema description (documentation only). |
| `custom` | a function path (repeatable) | A custom validator function. Repeat for several checks. |
| `nested` | flag (no value) | Validate a nested model or collection recursively. |
| `default` | flag (no value) | Use the type's `Default` when the field is absent from the body. |

Unknown rules are a compile error.

```rust
use tork::api_model;

#[api_model]
pub struct CreateOrder {
    #[field(min_length = 1, max_length = 120)]
    pub name: String,

    #[field(max_length = 300)]
    pub description: Option<String>,

    #[field(gt = 0)]
    pub price: f64,

    #[field(default)]
    pub tags: std::collections::HashSet<String>,
}
```

## Notes

- An `Option<T>` field is only checked when a value is present, so optional fields stay
  optional.
- A custom validator has the signature `fn(value: &T, ctx: &()) -> garde::Result`. It
  returns its own message with `garde::Error::new(...)`.
- `title` and `description` only affect the schema; they do not validate anything.
- The same `#[field(...)]` rules are also read by [`#[settings]`](/docs/framework/attributes/settings)
  and [`FormModel`](/docs/framework/attributes/forms), with small differences noted on
  those pages.
