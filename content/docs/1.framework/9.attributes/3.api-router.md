# `#[api_router]`

Marks a module that holds routes. It scans the module for handler functions (anything
annotated with `#[get]`, `#[post]`, `#[put]`, `#[patch]`, `#[delete]`, `#[sse]`,
`#[post_sse]`, or `#[websocket]`), in source order, and generates a `router()` function
inside the module that returns a `tork::Router` with all of them registered.

**Applies to:** an inline module (`pub mod name { ... }`). A file-only declaration like
`mod name;` is rejected; the module must have a body.

## Options

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| `prefix` | a string | none | A base path joined in front of every route in the module. |
| `tags` | a list of strings | empty | OpenAPI tags applied to every route in the module. |
| `throttle` | see [routes](/docs/framework/attributes/routes#throttle) | none | A default rate-limit policy for every route; a route's own `throttle` overrides it. |

Unknown options are a compile error.

```rust
use tork::{api_router, get};

#[api_router(prefix = "/users", tags = ["users"])]
pub mod users_router {
    use super::*;

    #[get("/{id}")]
    pub async fn show(id: i64) -> tork::Result<i64> {
        Ok(id)
    }
}

// The macro generates `users_router::router()`.
```

## Notes

- The `prefix` is joined with each route's local path, and a trailing slash is dropped
  except for the root path.
- `tags` are merged with any per-route tags, not replaced.
- A common convention is to re-export the generated function so callers write a short
  name: `pub use users_router::router;`.
