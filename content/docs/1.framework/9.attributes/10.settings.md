# `#[settings(...)]` and `#[setting(...)]`

`#[settings]` turns a struct into typed configuration. It generates a `load()` function
that reads values from defaults, config files, a `.env` file, the process environment, and
a secrets directory, then validates them. `#[setting(...)]` configures one field: its
default and its validation rules.

**Applies to:** a struct with named fields.

## `#[settings(...)]` options

| Option | Value | Effect |
| --- | --- | --- |
| `prefix` | a string | Environment variable prefix, for example `APP` reads `APP_*`. |
| `env_file` | a string | Path to a `.env` file to load. |
| `config_file` | a string | Path to a primary config file. |
| `files` | a list of strings | Additional config files, applied in order. |
| `secrets_dir` | a string | Directory of secret files, one file per setting. |

Unknown options are a compile error.

## `#[setting(...)]` options

| Option | Value | Effect |
| --- | --- | --- |
| `default` | flag, or `= expression` | Use `Default`, or the given expression, when the value is absent. A string literal is converted into the field type. |
| `min_length` | an integer | Minimum string length. |
| `max_length` | an integer | Maximum string length. |
| `ge` / `le` | a number | Inclusive numeric bounds. |
| `gt` / `lt` | a number | Exclusive numeric bounds. |
| `email` | flag | Must be a valid email address. |
| `custom` | a function path (repeatable) | A custom validator function. |
| `nested` | flag | Validate a nested settings group recursively. |
| `secret` | flag | Mark as a masked secret. Validation is skipped for secret fields. |

Unknown options are a compile error.

```rust
use tork::{settings, SecretString};

#[settings(prefix = "APP", env_file = ".env")]
pub struct AppConfig {
    #[setting(default = "my_api")]
    pub name: String,

    #[setting(default, ge = 1, le = 65535)]
    pub port: u16,

    #[setting(secret)]
    pub api_key: SecretString,
}

// AppConfig::load()? reads and validates the configuration.
```

## Notes

- `load()` runs validation as it loads, so invalid configuration fails fast at boot.
- A `Default` implementation is generated automatically when every field has a default and
  the struct has no generics, which lets it serve as an absent nested group.
- A `secret` field is masked in debug output; use `SecretString` for its type. A string
  literal `default` works for it because it is converted into the field type.
