# `#[tork::lifespan]`

Gives a [resources](/docs/framework/attributes/resources) container a startup and shutdown
lifecycle. The `startup` function builds the container when the application boots, and its
resources are registered for injection. The optional `shutdown` function releases them
when the application stops.

**Applies to:** an inherent `impl` block (not a trait `impl`). The type must also derive
`Resources`.

**Options:** none.

```rust
use tork::{LifespanContext, Logger, Resources, Result};

#[derive(Clone, Resources)]
pub struct AppState {
    #[resource]
    pub store: UserStore,
}

#[tork::lifespan]
impl AppState {
    async fn startup(_ctx: LifespanContext) -> Result<Self> {
        Ok(AppState { store: UserStore::seed() })
    }

    async fn shutdown(self) -> Result<()> {
        Logger::new("Lifecycle").info("stopped").emit();
        Ok(())
    }
}
```

## Rules

- The block may contain only `startup` and `shutdown`. Put any helper methods in a
  separate `impl` block.
- Both methods must be `async`. `startup` is required; `shutdown` is optional and defaults
  to doing nothing.
- `startup` returns `tork::Result<Self>`, so a failure (an unreachable database, for
  example) stops the application from starting.
- Register the lifespan on the application with `App::lifespan::<AppState>()`.
