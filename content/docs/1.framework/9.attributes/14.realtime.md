# `#[sse]`, `#[post_sse]`, and `#[websocket]`

These turn an `async fn` into a realtime route. `#[sse]` and `#[post_sse]` stream
Server-Sent Events; `#[websocket]` upgrades the connection to a WebSocket. Like the
ordinary route attributes, the path is the first, positional argument.

## `#[sse]` and `#[post_sse]`

The two are the same except for the default method: `#[sse]` is a `GET` route and
`#[post_sse]` is a `POST` route. The handler returns `tork::Result<Sse<T>>`.

**Applies to:** an `async fn`.

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| *(path)* | a string, positional and required | — | The route path. |
| `method` | a method name such as `POST` | `GET` for `#[sse]`, `POST` for `#[post_sse]` | Overrides the HTTP method. |
| `event` | a string | none | Default event name for the stream. |
| `response_model` | a type | none | The event data type for the OpenAPI document. |
| `summary` | a string | none | OpenAPI summary. |
| `description` | a string | none | OpenAPI description. |
| `tags` | a list of strings | empty | OpenAPI tags. |
| `throttle` | a policy | inherits | Per-route rate limit, with the same forms as on [routes](/docs/framework/attributes/routes#throttle). |

```rust
use tork::{sse, Sse};

#[sse("/events", event = "tick")]
async fn events() -> tork::Result<Sse<String>> {
    todo!()
}
```

Heartbeats, the retry hint, and connection limits are configured on the `Sse<T>` value you
return, not on the attribute.

## `#[websocket]`

Upgrades the connection to a WebSocket. It is always a `GET` route, and the handler returns
`tork::Result<()>`.

**Applies to:** an `async fn` that takes exactly one `WebSocket` parameter.

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| *(path)* | a string, positional and required | — | The route path. |
| `summary` | a string | none | OpenAPI summary. |
| `description` | a string | none | OpenAPI description. |
| `tags` | a list of strings | empty | OpenAPI tags. |
| `max_message_size` | a size like `"2KB"` | none | Largest accepted message. |
| `max_frame_size` | a size like `"1KB"` | none | Largest accepted frame. |
| `idle_timeout` | a duration like `"10s"` | none | Close the socket after this much inactivity. |
| `origins` | a list of strings | empty | Allowed `Origin` values for the handshake. |
| `allow_any_origin` | a bool | `false` | Accept a handshake from any origin. |
| `incoming` | a type | none | The inbound message type, for the AsyncAPI document. |
| `outgoing` | a type | none | The outbound message type, for the AsyncAPI document. |
| `throttle` | a policy | inherits | Per-route rate limit, as on [routes](/docs/framework/attributes/routes#throttle). |

```rust
use tork::{websocket, WebSocket};

#[websocket("/ws/{room}", idle_timeout = "10s", origins = ["https://app.example.com"])]
async fn chat(socket: WebSocket, room: String) -> tork::Result<()> {
    Ok(())
}
```

Other parameters work like ordinary handler parameters: a name matching a `{placeholder}`
is read from the path, and everything else is resolved by dependency injection.
