# `#[derive(FormModel)]`, `#[file(...)]`, and `#[form(...)]`

`FormModel` makes a struct readable from a `multipart/form-data` body, read in a handler
with `Multipart<T>`. File fields are marked with `#[file]` and text fields keep their
`#[field]` rules. The same `#[file]` and `#[form]` attributes can also go directly on
handler parameters when you prefer individual arguments over a struct.

**Applies to:** a struct with named fields.

```rust
use tork::{FileBytes, FormModel};

#[derive(FormModel)]
pub struct ProfileForm {
    #[file(max_size = "2MB", content_types = ["image/png", "image/jpeg"], sniff = true)]
    pub avatar: FileBytes,

    #[field(min_length = 2, max_length = 40)]
    pub display_name: String,
}
```

A field is a file field if it carries `#[file]` or its type is `FileBytes` or `UploadFile`.
File fields may be wrapped in `Option<...>` for an optional file or `Vec<...>` for several
files under the same name.

## `#[file(...)]` options

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| `name` | a string | the field or parameter name | The multipart field name to read. |
| `max_size` | a size like `"2MB"` | none | Largest allowed file. `max_size_each` is an alias. |
| `content_types` | a list of strings | empty | Allowed MIME types, for example `["image/png"]`. |
| `sniff` | a bool, or the bare flag (which means `true`) | `false` | Check the file's real bytes against `content_types`, so a file cannot lie about its type. |

## `#[form(...)]` options

| Option | Value | Effect |
| --- | --- | --- |
| `name` | a string | The form field name to read for a text field. |

## `#[field(...)]` on form fields

Text fields honor the length and numeric rules from [`#[field]`](/docs/framework/attributes/api-model):
`min_length`, `max_length`, `ge`, `le`, `gt`, `lt`. The documentation-only rules `title`
and `description` are ignored for forms.

## On handler parameters

The same attributes work on individual parameters, which puts a route in multipart mode
without a struct.

```rust
use tork::{post, FileBytes};

#[post("/upload")]
async fn upload(
    #[file(content_types = ["image/png"])] avatar: FileBytes,
    #[form(name = "title")] title: String,
) -> tork::Result<usize> {
    Ok(avatar.len())
}
```

A failing form returns `422` with one entry per offending field. Route-level limits such
as `max_file_size` are set with the [`upload(...)`](/docs/framework/attributes/routes#upload)
option on the route.
