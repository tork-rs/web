# Route methods: `#[get]`, `#[post]`, `#[put]`, `#[patch]`, `#[delete]`

Each of these turns an `async fn` into a route for one HTTP method. They all accept the
same options. The path is the first, positional argument; everything else is a named
option.

**Applies to:** an `async fn`.

```rust
use tork::get;

#[get("/users/{id}")]
async fn show(id: i64) -> tork::Result<i64> {
    Ok(id)
}
```

## Options

| Option | Value | Default | Effect |
| --- | --- | --- | --- |
| *(path)* | a string, positional and required | — | The route path. Use `{name}` for a path parameter and `{*rest}` for a wildcard. |
| `response_model` | a type | the handler return type | The success response type for the OpenAPI schema. When it differs from the return type, the value is converted into it. |
| `summary` | a string | none | Short OpenAPI summary. |
| `description` | a string | none | Longer OpenAPI description. |
| `status_code` | an integer or a `StatusCode` | `200` | The success status. An integer literal such as `201` is accepted directly. |
| `tags` | a list of strings | empty | OpenAPI tags for this route, in addition to the router tags. |
| `on_request` | a function path (repeatable) | none | Scoped hook run when the request arrives. |
| `on_response` | a function path (repeatable) | none | Scoped hook run before the response is sent. |
| `on_error` | a function path (repeatable) | none | Scoped hook run when the handler returns an error. |
| `on_validation_error` | a function path (repeatable) | none | Scoped hook run when body validation fails. |
| `upload(...)` | a group of upload limits | none | Per-route limits for multipart uploads. See below. |
| `throttle` | a policy | inherits | Per-route rate limit. See below. |

Unknown options are a compile error.

## Handler rules

- A handler cannot take `self`.
- Each parameter must be a simple identifier (no destructuring).
- A handler may declare at most one request body. `Valid<T>`, `Json<T>`, `Form<T>`, and
  `Multipart<T>` each consume the body, so only one of them may appear.
- A parameter whose name matches a `{placeholder}` in the path is read from the path;
  every other parameter is resolved by dependency injection.

## `upload(...)`

A group of limits that apply when the handler reads a multipart body (with `#[file]`
or `#[form]` parameters). All keys are optional.

| Key | Value | Effect |
| --- | --- | --- |
| `max_body_size` | a size like `"5MB"` | Largest allowed request body. |
| `max_file_size` | a size like `"2MB"` | Largest allowed single file. |
| `memory_threshold` | a size like `"512KB"` | Above this, a file spools to disk instead of memory. |
| `max_files` | an integer | Largest number of files accepted. |

Sizes accept `B`, `KB`, `MB` (and the `KiB`/`MiB` forms); a bare number is bytes.

```rust
#[post("/avatars", upload(max_files = 1, max_file_size = "2MB"))]
async fn upload(#[file] avatar: tork::FileBytes) -> tork::Result<usize> {
    Ok(avatar.len())
}
```

## `throttle`

Applies a rate-limit policy to the route. It has several forms:

| Form | Meaning |
| --- | --- |
| `throttle = "skip"` | Bypass rate limiting for this route. |
| `throttle = "name"` | Apply the named policy defined with `App::throttle`. |
| `throttle = ["short", "long"]` | Apply several named policies. |
| `throttle(name = "api")` | The named policy, written as a group. |
| `throttle(limit = 5, ttl = 60)` | An inline policy: 5 requests per 60 seconds. |
| `throttle(name = "api", key = ByUserId)` | A policy with a custom key type. |

The group form requires either `name`, or both `limit` and `ttl`. A route's own
`throttle` overrides a default set on the [router](/docs/framework/attributes/api-router).

```rust
#[post("/login", throttle(limit = 5, ttl = 60))]
async fn login() -> tork::Result<&'static str> {
    Ok("ok")
}
```
