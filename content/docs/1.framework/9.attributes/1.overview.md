# Attributes Overview

Tork is driven by attributes. You annotate a function, a module, or a struct, and a
macro generates the wiring behind it. This section is a reference: one page per
attribute, listing what it applies to, every option it accepts, and a minimal
example. For task focused guides, see the rest of the documentation; this section is
the exhaustive list.

There are three kinds of attribute.

## Item macros

These go on a function, a module, or an `impl` block.

| Attribute | Goes on | Purpose |
| --- | --- | --- |
| [`#[tork::main]`](/docs/framework/attributes/main) | `async fn main` | Starts the async runtime. |
| [`#[api_router]`](/docs/framework/attributes/api-router) | a module | Collects the routes inside into a `router()`. |
| [`#[get]` `#[post]` `#[put]` `#[patch]` `#[delete]`](/docs/framework/attributes/routes) | `async fn` | Turns a function into a route. |
| [`#[sse]` `#[post_sse]` `#[websocket]`](/docs/framework/attributes/realtime) | `async fn` | Streaming and socket routes. |
| [`#[api_model]`](/docs/framework/attributes/api-model) | a struct | A validated, documented request or response model. |
| [`#[settings]`](/docs/framework/attributes/settings) | a struct | Typed configuration loaded from the environment. |
| [`#[tork::dependency]`](/docs/framework/attributes/dependency) | an `impl` block | A type that builds itself from the request. |
| [`#[tork::lifespan]`](/docs/framework/attributes/lifespan) | an `impl` block | Startup and shutdown for a resource container. |
| [`#[middleware]`](/docs/framework/attributes/middleware) | `async fn` | A request middleware layer. |

## Derive macros

These go in a `#[derive(...)]` list on a struct or enum.

| Derive | Goes on | Purpose |
| --- | --- | --- |
| [`Resources`](/docs/framework/attributes/resources) | a struct | Registers fields as injectable resources. |
| [`Inject`](/docs/framework/attributes/inject) | a struct | Builds a service by resolving each field. |
| [`FormModel`](/docs/framework/attributes/forms) | a struct | A multipart form, including file fields. |
| [`AppError`](/docs/framework/attributes/app-error) | a struct or enum | Maps your error type to an HTTP response. |

## Field and parameter attributes

These go on a struct field, or on a handler parameter, and belong to one of the macros above.

| Attribute | Goes on | Belongs to |
| --- | --- | --- |
| [`#[field(...)]`](/docs/framework/attributes/api-model) | a model field | `#[api_model]`, `FormModel`, `#[settings]` |
| [`#[resource]`](/docs/framework/attributes/resources) | a struct field | `Resources` |
| [`#[inject(...)]` `#[logger(...)]`](/docs/framework/attributes/inject) | a struct or field | `Inject` |
| [`#[file(...)]` `#[form(...)]`](/docs/framework/attributes/forms) | a field or parameter | `FormModel`, route handlers |
| [`#[setting(...)]`](/docs/framework/attributes/settings) | a field | `#[settings]` |
| [`#[status(...)]`](/docs/framework/attributes/app-error) | a type | `AppError` |
