# `#[tork::dependency]`

Makes a type into a dependency: a value that knows how to build itself from the request.
You write an async `resolve` function, and the macro generates the `FromRequest`
implementation that runs it. Any handler can then ask for the type as an argument.

**Applies to:** an `impl` block that contains an async `fn resolve`.

**Options:** none.

```rust
use tork::{BearerToken, Error};

#[derive(Clone)]
pub struct CurrentUser {
    pub id: i64,
}

#[tork::dependency]
impl CurrentUser {
    async fn resolve(token: BearerToken) -> tork::Result<Self> {
        let id = token
            .token()
            .parse::<i64>()
            .map_err(|_| Error::unauthorized("invalid token"))?;
        Ok(CurrentUser { id })
    }
}
```

## Rules

- The `impl` block must contain an async function named `resolve` that returns
  `tork::Result<Self>`.
- The parameters of `resolve` are themselves dependencies. They are resolved in order,
  each through `FromRequest`, before `resolve` runs. This is what lets dependencies build
  on other dependencies.
- `resolve` cannot take `self`, and its parameters must be simple identifiers.
- Because `resolve` returns a `Result`, a dependency can stop the request by returning
  an error, which makes it a request guard.
