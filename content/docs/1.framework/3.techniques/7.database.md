# Database

Tork integrates with `tork-orm` through the standard resource system: connect at startup in a lifespan, register the pool as `Arc<Database>`, and declare it as a parameter in any handler or `#[derive(Inject)]` struct.

## Setup

Add `tork-orm` to `Cargo.toml` with the backend feature and the `tork` integration flag:

```toml
[dependencies]
tork     = { git = "https://github.com/tork-rs/tork-framework.git" }
tork-orm = { git = "https://github.com/tork-rs/tork-orm.git", default-features = false, features = ["sqlite", "tork", "migrations"] }
tokio    = { version = "1", features = ["rt-multi-thread", "macros"] }
```

For PostgreSQL replace `sqlite` with `postgres`; for MySQL use `mysql`.

If you created your project with `tork new myapp --db sqlite`, these dependencies are already in place.

## Define the database settings

Use `#[settings]` to load the connection URL and pool size from environment variables:

```rust
// src/core/db.rs
use tork::settings;

#[settings(prefix = "DB")]
pub struct DatabaseConfig {
    /// DATABASE_URL or DB_URL
    #[setting(default = "sqlite://app.db")]
    pub url: String,
    /// DB_MAX_CONNECTIONS
    #[setting(default = 10, ge = 1, le = 64)]
    pub max_connections: u32,
}
```

Add defaults to `.env`:

```
DB_URL=sqlite://app.db
DB_MAX_CONNECTIONS=10
```

## Create the database resource

Wrap `Arc<Database>` in a `#[derive(Resources)]` struct and open the connection in the lifespan `startup` hook:

```rust
// src/core/db.rs (continued)
use std::sync::Arc;
use tork::{lifespan, LifespanContext, Resources, Result};
use tork_orm::prelude::*;

#[derive(Clone, Resources)]
pub struct Db {
    #[resource]
    pub database: Arc<Database>,
}

#[lifespan]
impl Db {
    async fn startup(_ctx: LifespanContext) -> Result<Self> {
        let config = DatabaseConfig::load()?;

        let database = Database::connect(&config.url, config.max_connections).await?;

        // Run pending migrations on startup.
        database.run_migrations("migrations/").await?;

        Ok(Db { database: Arc::new(database) })
    }

    async fn shutdown(self) -> Result<()> {
        // Return all connections to the OS.
        self.database.close().await;
        Ok(())
    }
}
```

## Register the lifespan

```rust
// src/main.rs
mod core;
mod models;
mod repositories;
mod routers;
mod schemas;
mod services;

use tork::{App, OpenApi};

#[tork::main]
async fn main() -> tork::Result<()> {
    let config = core::settings::AppConfig::load()?;

    App::new()
        .lifespan::<core::db::Db>()   // connects DB, runs migrations, closes on shutdown
        .include_router(routers::router())
        .openapi(
            OpenApi::new()
                .title(&config.name)
                .version("0.1.0")
                .json("/openapi.json")
                .docs("/docs"),
        )
        .serve(&config.host)
        .await
}
```

## Inject Arc<Database> directly into a handler

Once the lifespan is registered, any handler can declare `Arc<Database>` as a parameter and receive the pool:

```rust
use std::sync::Arc;
use tork::{get, Result};
use tork_orm::prelude::*;

use crate::models::User;

#[get("/users", response_model = Vec<User>)]
pub async fn list_users(db: Arc<Database>) -> Result<Vec<User>> {
    let users = User::query()
        .filter(User::is_active.eq(true))
        .order_by(User::id.asc())
        .all(&*db)
        .await?;
    Ok(users)
}

#[get("/users/{id}", response_model = User)]
pub async fn get_user(id: i64, db: Arc<Database>) -> Result<User> {
    let user = User::find(&*db, id).await?;
    Ok(user)
}
```

`Arc<Database>` is cheap to clone; each handler invocation gets a clone of the pointer, not a new connection.

## Inject through a repository

For real applications, put the query logic in a repository and inject that instead. Repositories are cleaner to test and keep handlers focused on HTTP concerns:

```rust
// src/repositories/user_repo.rs
use std::sync::Arc;
use tork::Inject;
use tork_orm::prelude::*;

use crate::models::User;

#[derive(Inject)]
pub struct UserRepository {
    db: Arc<Database>,
}

impl UserRepository {
    pub async fn list_active(&self) -> tork_orm::Result<Vec<User>> {
        User::query()
            .filter(User::is_active.eq(true))
            .order_by(User::username.asc())
            .all(&*self.db)
            .await
    }

    pub async fn create(&self, username: String, email: String) -> tork_orm::Result<User> {
        User::create(
            &*self.db,
            &User { id: 0, username, email, is_active: true },
        )
        .await
    }
}
```

```rust
// src/routers/users.rs
use tork::{get, post, Valid};
use serde::Deserialize;

use crate::repositories::user_repo::UserRepository;
use crate::models::User;

#[derive(Deserialize, garde::Validate)]
pub struct CreateUserBody {
    #[garde(length(min = 2, max = 50))]
    pub username: String,
    #[garde(email)]
    pub email: String,
}

#[get("/users", response_model = Vec<User>)]
pub async fn list_users(repo: UserRepository) -> tork::Result<Vec<User>> {
    Ok(repo.list_active().await?)
}

#[post("/users", response_model = User, status_code = 201)]
pub async fn create_user(
    repo: UserRepository,
    body: Valid<CreateUserBody>,
) -> tork::Result<User> {
    let b = body.into_inner();
    Ok(repo.create(b.username, b.email).await?)
}
```

## Transactions in a handler

Pass `Arc<Database>` to the transaction API directly from a handler or a repository:

```rust
use std::sync::Arc;
use tork::{post, Result};
use tork_orm::prelude::*;
use tork_orm::transaction;

use crate::models::{Order, OrderItem};

#[post("/orders", response_model = Order, status_code = 201)]
pub async fn create_order(db: Arc<Database>) -> Result<Order> {
    let order = transaction!(db, |tx| async move {
        let order = Order::create(tx, &Order { id: 0, total_cents: 9900 }).await?;
        OrderItem::create(tx, &OrderItem { id: 0, order_id: order.id, qty: 1 }).await?;
        Ok(order)
    })
    .await?;

    Ok(order)
}
```

If either insert fails, the whole transaction is rolled back. See [Transactions](/docs/orm/querying/transactions) for the complete API.

## Environment configuration

For the generated project the database URL lives in `.env`:

```
DB_URL=sqlite://app.db
DB_MAX_CONNECTIONS=10
```

For PostgreSQL:

```
DB_URL=postgres://user:pass@localhost:5432/mydb
DB_MAX_CONNECTIONS=20
```

`tork-orm` reads the URL at startup when `Database::connect` is called. There is no additional parsing layer.

## Running migrations

The `startup` hook above calls `database.run_migrations("migrations/")` to apply any pending migrations before the server accepts requests. Alternatively, run them with the CLI before deploying:

```bash
tork migrate up
```

See [Migrations](/docs/orm/schema/migrations) for the complete CLI reference.

## Testing with an in-memory database

For tests, connect to `sqlite::memory:` and set up the schema directly. The `#[derive(Inject)]` mechanism supports overriding resources with test doubles:

```rust
use tork::testing::TestClient;
use tork_orm::prelude::*;

use crate::models::User;

async fn test_db() -> Arc<Database> {
    let db = Database::connect("sqlite::memory:", 1).await.unwrap();
    db.execute(
        "CREATE TABLE users (
            id INTEGER PRIMARY KEY,
            username TEXT NOT NULL UNIQUE,
            email TEXT NOT NULL,
            is_active INTEGER NOT NULL
        )"
        .into(),
        vec![],
    )
    .await
    .unwrap();
    Arc::new(db)
}

#[tokio::test]
async fn test_list_users() {
    let db = test_db().await;

    User::create(
        &*db,
        &User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
    )
    .await
    .unwrap();

    let client = TestClient::new(app_with_db(db.clone())).await;
    let resp = client.get("/users").send().await;
    assert_eq!(resp.status(), 200);
    let users: Vec<serde_json::Value> = resp.json().await;
    assert_eq!(users.len(), 1);
}
```

For the full testing API, see [Testing](/docs/framework/fundamentals/testing).
