# Database

Tork keeps database access in the same resource system as the rest of the app. Create the pool in a lifespan, register it as `Arc<Database>`, and inject it where you need it. The ORM still owns SQL, transactions, and migrations.

## Setup

Add the framework and ORM crates with the integration feature enabled.

```toml
[dependencies]
tork = { git = "https://github.com/tork-rs/tork-framework.git" }
tork-orm = { git = "https://github.com/tork-rs/tork-orm.git", default-features = false, features = ["sqlite", "tork", "migrations"] }
tokio = { version = "1", features = ["rt-multi-thread", "macros"] }
```

Swap `sqlite` for `postgres` or `mysql` when you enable those backends.

The generated app reads `DB_URL` and `DB_MAX_CONNECTIONS` from settings.

```
DB_URL=sqlite::memory:
DB_MAX_CONNECTIONS=5
```

`sqlite::memory:` keeps the example self-contained. Point `DB_URL` at `sqlite://app.db` when you want the data on disk.

## Project Layout

A small app usually keeps the database setup in `src/db.rs`, the schema in `src/models.rs`, the shared query logic in `src/repositories.rs`, the route handlers in `src/routers.rs`, and the entrypoint in `src/main.rs`.

::code-group

```rust [src/db.rs]
use std::sync::Arc;

use tork::{settings, LifespanContext, Resources, Result};
use tork_orm::migration::FileMigrator;
use tork_orm::prelude::*;

#[settings(prefix = "DB")]
pub struct DatabaseConfig {
    #[setting(default = "sqlite::memory:")]
    pub url: String,
    #[setting(default = 5, ge = 1, le = 64)]
    pub max_connections: u32,
}

#[derive(Clone, Resources)]
pub struct Db {
    #[resource]
    pub database: Arc<Database>,
}

#[tork::lifespan]
impl Db {
    async fn startup(_ctx: LifespanContext) -> Result<Self> {
        let config = DatabaseConfig::load()?;
        let database = Database::connect(&config.url, config.max_connections).await?;
        FileMigrator::new(
            database.clone(),
            concat!(env!("CARGO_MANIFEST_DIR"), "/migrations"),
        )
        .up()
        .await?;

        Ok(Db {
            database: Arc::new(database),
        })
    }

    async fn shutdown(self) -> Result<()> {
        self.database.close().await;
        Ok(())
    }
}
```

```rust [src/models.rs]
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50), unique)]
    pub username: String,
    #[field(varchar(length = 255), unique)]
    pub email: String,
    pub is_active: bool,
}
```

```rust [src/schemas.rs]
use tork::api_model;

use crate::models::User;

#[api_model]
pub struct UserOut {
    pub id: i64,
    pub username: String,
    pub email: String,
    pub is_active: bool,
}

impl From<User> for UserOut {
    fn from(user: User) -> Self {
        Self {
            id: user.id,
            username: user.username,
            email: user.email,
            is_active: user.is_active,
        }
    }
}

#[api_model]
pub struct CreateUserInput {
    #[field(min_length = 2, max_length = 50)]
    pub username: String,
    #[field(min_length = 3, max_length = 255)]
    pub email: String,
}
```

```rust [src/repositories.rs]
use std::sync::Arc;

use tork::Inject;
use tork_orm::prelude::*;

use crate::models::User;
use crate::schemas::{CreateUserInput, UserOut};

#[derive(Inject)]
pub struct UserRepository {
    db: Arc<Database>,
}

impl UserRepository {
    pub async fn list_active(&self) -> Result<Vec<UserOut>> {
        Self::list_active_on(&self.db).await
    }

    pub async fn list_active_on(executor: impl Executor) -> Result<Vec<UserOut>> {
        let users = User::query()
            .filter(User::is_active.eq(true))
            .order_by(User::username.asc())
            .all(executor)
            .await?;

        Ok(users.into_iter().map(UserOut::from).collect())
    }

    pub async fn create(&self, input: CreateUserInput) -> Result<UserOut> {
        let user = User::create(
            &self.db,
            &User {
                id: 0,
                username: input.username,
                email: input.email,
                is_active: true,
            },
        )
        .await?;

        Ok(UserOut::from(user))
    }
}
```

```rust [src/routers.rs]
use std::sync::Arc;

use tork::{api_router, get, post, Valid};
use tork_orm::prelude::Database;

use crate::models::User;
use crate::repositories::UserRepository;
use crate::schemas::{CreateUserInput, UserOut};

#[api_router(prefix = "/users", tags = ["users"])]
pub mod users_router {
    use super::*;

    #[get("/", response_model = Vec<UserOut>, summary = "List users")]
    pub async fn list_users(repo: UserRepository) -> tork::Result<Vec<UserOut>> {
        Ok(repo.list_active().await?)
    }

    #[get("/{user_id}", response_model = UserOut, summary = "Get user by id")]
    pub async fn get_user(user_id: i64, db: Arc<Database>) -> tork::Result<UserOut> {
        let user = User::find(&*db, user_id).await?;
        Ok(UserOut::from(user))
    }

    #[post("/", response_model = UserOut, status_code = 201, summary = "Create user")]
    pub async fn create_user(
        repo: UserRepository,
        body: Valid<CreateUserInput>,
    ) -> tork::Result<UserOut> {
        Ok(repo.create(body.into_inner()).await?)
    }
}

pub use users_router::router;
```

```rust [src/main.rs]
mod db;
mod models;
mod repositories;
mod routers;
mod schemas;

use tork::App;

use crate::db::Db;
use crate::routers::users;

#[tork::main]
async fn main() -> tork::Result<()> {
    App::new()
        .lifespan::<Db>()
        .include_router(users::router())
        .serve("0.0.0.0:8000")
        .await
}
```

::

After startup, any handler or injected service can ask for `Arc<Database>` directly. Cloning it is cheap, because the pool stays shared.

## Reusable Query Helpers

When the same query logic should work on the pool and inside a transaction, make the helper accept `impl Executor`.

```rust
use tork_orm::prelude::*;

use crate::models::User;
use crate::schemas::UserOut;

pub async fn list_active_users(executor: impl Executor) -> Result<Vec<UserOut>> {
    let users = User::query()
        .filter(User::is_active.eq(true))
        .order_by(User::username.asc())
        .all(executor)
        .await?;

    Ok(users.into_iter().map(UserOut::from).collect())
}
```

`Database`, `Arc<Database>`, and `Transaction` all implement `Executor`, so the same helper works in a handler, a repository, or a transaction block.

## Transactions

For multi-step writes, wrap the work in `transaction!` and keep using the same database handle.

```rust
use tork_orm::{prelude::*, transaction};

let created = transaction!(db, |tx| async move {
    let user = User::create(
        tx,
        &User {
            id: 0,
            username: "alice".into(),
            email: "alice@example.com".into(),
            is_active: true,
        },
    )
    .await?;

    Ok(user)
})
.await?;
```

The same `tx` can be passed to any helper that accepts `impl Executor`. The full transaction API lives in [Transactions](/docs/orm/querying/transactions).

## Migrations and Tests

`FileMigrator::new(database.clone(), "migrations").up().await?` applies SQL files at startup. If you prefer to run migrations before deploy, keep the same `Db` lifespan and use the ORM CLI from the migrations chapter.

For tests, build the app with `App::build_test()` and override the `Arc<Database>` resource with a fresh database. The framework testing guide shows the full client flow and the resource override pattern.
