# Caching

Tork ships a small, store-agnostic cache with TTL support. It is a manual key/value API: you decide what to cache and when. The same handler code works whether values are kept in memory or in Redis, so you can start in memory and swap the backend later without changing handlers.

## Enabling the cache

Configure a cache on the app, then inject it where you need it. The default backend is an in-memory store.

```rust
use tork::{App, Cache};

App::new().cache(Cache::in_memory());
```

Caching is opt-in. Without `App::cache(...)`, injecting a `Cache` fails with a clear error.

## Reading and writing

Inject `Cache` by type. Values are serialized to JSON, so any `serde` type can be cached. Keys are `&str`.

```rust
use tork::{get, Cache};

#[get("/greeting")]
pub async fn greeting(cache: Cache) -> tork::Result<String> {
    cache.set("greeting", &"hello").await?;
    let value: Option<String> = cache.get("greeting").await?;
    Ok(value.unwrap_or_default())
}
```

The methods:

| Method | Effect |
| --- | --- |
| `get::<T>(key)` | Returns the value, or `None` if absent or expired |
| `set(key, &value)` | Stores the value with the cache's default TTL |
| `set_ttl(key, &value, ttl)` | Stores the value, expiring it after `ttl` |
| `get_or_set(key, ttl, init)` | Returns the cached value, or computes, stores, and returns it |
| `delete(key)` | Removes one entry |
| `clear()` | Removes every entry |

`get::<T>` needs `T: DeserializeOwned`, and `set` needs `T: Serialize`. `get` returns `Ok(None)` when the key is absent or has expired.

## Cache-aside in one call

`get_or_set` is the common pattern: return the cached value on a hit, or run the closure once on a miss and store its result. The `ttl` argument is an `Option<Duration>`, where `None` uses the cache's default.

```rust
use tork::{get, Cache};

#[get("/users/{id}")]
pub async fn get_user(id: i64, users: UserService, cache: Cache) -> tork::Result<User> {
    let user = cache
        .get_or_set(&format!("user:{id}"), None, || async move { users.load(id).await })
        .await?;
    Ok(user)
}
```

The closure runs only on a miss, so a slow lookup happens at most once per key until the entry expires. The value type needs both `Serialize` and `DeserializeOwned`.

## Time-to-live

Set a per-entry TTL with `set_ttl`, or a default for the whole cache with `default_ttl`.

```rust
use std::time::Duration;

// Expire this entry after 30 seconds.
cache.set_ttl("token", &token, Duration::from_secs(30)).await?;

// Apply a default to every plain `set` and to `get_or_set(.., None, ..)`.
let cache = Cache::in_memory().default_ttl(Duration::from_secs(60));
```

A zero `Duration` means never expire. With no TTL set, entries live until the in-memory store reaches its capacity (10,000 entries by default) and evicts the least-used ones.

## Swapping the store

The backend is a `CacheStore`: a small trait with `get`, `set`, `delete`, and `clear` over byte values. The built-in `MemoryStore` is the default. Supply your own with `Cache::new(store)`.

```rust
use tork::Cache;

let cache = Cache::new(MyCustomStore::new());
```

Because handlers only see the `Cache` handle, switching stores does not change them.

## Backing the cache with Redis

Enable the `redis` feature to share the cache across instances, so a value cached by one process is visible to the others.

```toml
tork = { version = "0.1", features = ["redis"] }
```

`Cache::redis(url)` opens its own connection, while `Cache::from_redis(&redis)` reuses one you already have.

```rust
use tork::{App, Cache, Redis};

let redis = Redis::connect("redis://127.0.0.1:6379").await?;

App::new()
    .redis(redis.clone())
    .cache(Cache::from_redis(&redis)); // shares the same connection
```

Keys are namespaced with a `tork:` prefix, so `clear()` scans and removes only this cache's keys and never flushes the whole database. For connecting, sharing one handle, and running raw commands, see [Redis](/docs/framework/techniques/redis).

## A complete example

A handler that caches a computed value with `get_or_set`, and a test that calls it.

::code-group

```rust [src/routers.rs]
use tork::{api_model, api_router, get, Cache};

#[api_model]
pub struct Square {
    pub n: i64,
    pub result: i64,
}

#[api_router(tags = ["math"])]
pub mod math_router {
    use super::*;

    #[get("/square/{n}")]
    pub async fn square(n: i64, cache: Cache) -> tork::Result<Square> {
        let value = cache
            .get_or_set(&format!("square:{n}"), None, || async move {
                Ok(Square { n, result: n * n })
            })
            .await?;
        Ok(value)
    }
}

pub use math_router::router;
```

```rust [src/lib.rs]
pub mod routers;
```

```rust [src/main.rs]
use tork::{App, Cache};

use myapp::routers;

#[tork::main]
async fn main() -> tork::Result<()> {
    App::new()
        .cache(Cache::in_memory())
        .include_router(routers::router())
        .serve("0.0.0.0:8000")
        .await
}
```

```rust [tests/cache.rs]
use tork::testing::TestClient;
use tork::{App, Cache};

use myapp::routers::{router, Square};

#[tokio::test]
async fn computes_and_caches() -> tork::Result<()> {
    let app = App::new().cache(Cache::in_memory()).include_router(router());
    let client = TestClient::new(app.build_test().await?).await?;

    let res = client.get("/square/9").send().await?;
    assert_eq!(res.status(), 200);
    assert_eq!(res.json::<Square>().await?.result, 81);

    Ok(())
}
```
