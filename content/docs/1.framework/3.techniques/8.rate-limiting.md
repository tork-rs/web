# Rate Limiting

Tork has a built-in throttler. You define named policies once on the app, then apply them with a `throttle` attribute on routes and routers. A request over the limit gets back `429 Too Many Requests` with a `Retry-After` header and the error code `RATE_LIMITED`.

Counts are tracked per route and per key. The key is the client IP by default, so each caller gets their own budget on each endpoint. Without `App::throttle(...)`, every `throttle` attribute is a no-op, so you can annotate routes first and turn limiting on later.

## Defining policies

Configure the throttler on the app. A policy has a name, a request limit, and a window in seconds. Mark one policy as the default to apply it to every route that does not set its own.

```rust
use tork::{App, Throttle};

App::new().throttle(
    Throttle::new()
        .policy("default", 100, 60) // 100 requests per 60s
        .policy("strict", 5, 60)    // 5 requests per 60s
        .default("default"),
);
```

The limit is a `u32` and the window is in whole seconds. A window of `0` is treated as `1`.

## Applying a policy

Add the `throttle` attribute to a route. It takes a named policy, an inline limit, or `"skip"`.

```rust
use tork::get;

// Use a named policy defined on the app.
#[get("/feed", throttle = "strict")]
pub async fn feed() -> tork::Result<&'static str> {
    Ok("feed")
}

// Set the limit and window inline, no named policy needed.
#[get("/search", throttle(limit = 3, ttl = 60))]
pub async fn search() -> tork::Result<&'static str> {
    Ok("results")
}

// Opt a route out of limiting.
#[get("/health", throttle = "skip")]
pub async fn health() -> tork::Result<&'static str> {
    Ok("ok")
}
```

`ttl` is the window in seconds. A named policy and an inline limit cannot be mixed in the same attribute: use one form or the other.

## A full example

This app sets a global default of 100 requests per minute, then gives one endpoint a tighter limit counted per API key instead of per IP. The custom key, the routes, the wiring, and a test each live in their own file.

::code-group

```rust [src/throttling.rs]
use tork::{RequestContext, ThrottleKey};

// Count requests per API key (the `x-api-key` header) instead of per IP.
pub struct ByApiKey;

impl ThrottleKey for ByApiKey {
    async fn throttle_key(ctx: &RequestContext) -> tork::Result<String> {
        let key = ctx
            .headers()
            .get("x-api-key")
            .and_then(|value| value.to_str().ok())
            .unwrap_or("anonymous");
        Ok(key.to_string())
    }
}
```

```rust [src/routers.rs]
use tork::{api_model, api_router, get};

use crate::throttling::ByApiKey;

#[api_model]
pub struct Quote {
    pub symbol: String,
    pub price: f64,
}

#[api_router(prefix = "/api", tags = ["market"])]
pub mod market_router {
    use super::*;

    // Inherits the global default policy (100 per minute).
    #[get("/status")]
    pub async fn status() -> tork::Result<&'static str> {
        Ok("ok")
    }

    // A tighter limit, counted per API key. This overrides the global default.
    #[get("/quote/{symbol}", throttle(limit = 5, ttl = 60, key = ByApiKey))]
    pub async fn quote(symbol: String) -> tork::Result<Quote> {
        Ok(Quote { symbol, price: 42.0 })
    }

    // Never limited.
    #[get("/health", throttle = "skip")]
    pub async fn health() -> tork::Result<&'static str> {
        Ok("ok")
    }
}

pub use market_router::router;
```

```rust [src/lib.rs]
pub mod routers;
pub mod throttling;
```

```rust [src/main.rs]
use tork::{App, Throttle};

use myapp::routers;

#[tork::main]
async fn main() -> tork::Result<()> {
    App::new()
        .throttle(
            Throttle::new()
                .policy("default", 100, 60)
                .default("default"),
        )
        .include_router(routers::router())
        .serve("0.0.0.0:8000")
        .await
}
```

```rust [tests/rate_limit.rs]
use tork::testing::TestClient;
use tork::{App, Throttle};

use myapp::routers;

async fn client() -> tork::Result<TestClient> {
    let app = App::new()
        .throttle(Throttle::new().policy("default", 100, 60).default("default"))
        .include_router(routers::router());
    Ok(TestClient::new(app.build_test().await?).await?)
}

#[tokio::test]
async fn quote_blocks_after_five_per_key() -> tork::Result<()> {
    let client = client().await?;

    // Five requests with the same API key succeed.
    for _ in 0..5 {
        let res = client.get("/api/quote/ACME").header("x-api-key", "k1").send().await?;
        assert_eq!(res.status(), 200);
    }

    // The sixth is blocked with 429 and a Retry-After header.
    let blocked = client.get("/api/quote/ACME").header("x-api-key", "k1").send().await?;
    assert_eq!(blocked.status(), 429);
    assert!(blocked.headers().get("retry-after").is_some());

    // A different key has its own budget.
    let other = client.get("/api/quote/ACME").header("x-api-key", "k2").send().await?;
    assert_eq!(other.status(), 200);

    Ok(())
}
```

::

Run the app with `cargo run`, then watch the limit kick in:

```bash
# The sixth call in a minute with the same key returns 429.
for i in $(seq 1 6); do
  curl -s -o /dev/null -w "%{http_code}\n" \
    -H 'x-api-key: k1' http://localhost:8000/api/quote/ACME
done
```

The first five print `200`, the sixth prints `429`.

## Router and endpoint precedence

A `throttle` on `#[api_router]` applies to every route inside it. A route can override the router with its own `throttle`, and `"skip"` opts out. The order is endpoint first, then router, then the global default.

```rust
use tork::{api_router, get};

#[api_router(prefix = "/orders", tags = ["orders"], throttle = "strict")]
pub mod orders_router {
    use super::*;

    // Inherits "strict" from the router.
    #[get("/")]
    pub async fn list() -> tork::Result<&'static str> {
        Ok("orders")
    }

    // Overrides the router with a tighter inline limit.
    #[get("/export", throttle(limit = 1, ttl = 60))]
    pub async fn export() -> tork::Result<&'static str> {
        Ok("export")
    }

    // Opts out of the router policy entirely.
    #[get("/health", throttle = "skip")]
    pub async fn health() -> tork::Result<&'static str> {
        Ok("ok")
    }
}
```

## Several limits at once

List more than one named policy to enforce a per-second and a per-minute cap together. A request has to satisfy every one, and the first to deny wins.

```rust
// Defined on the app:
// Throttle::new().policy("burst", 5, 1).policy("steady", 100, 60)
#[get("/upload", throttle = ["burst", "steady"])]
pub async fn upload() -> tork::Result<&'static str> {
    Ok("uploaded")
}
```

## Custom keys

Requests are counted per key. The default key is the client IP, taken from the peer socket address. That is the direct connection, so behind a load balancer or reverse proxy every request looks like it comes from the proxy. When you run behind one, key on something the proxy forwards instead, such as an API key, a user id, or a header it sets.

Implement `ThrottleKey` on a unit type and pass it with `key = ...`. The extractor is async and receives the full `RequestContext`, so it can read headers or resolve a dependency.

```rust
use tork::{RequestContext, ThrottleKey};

pub struct ByForwardedIp;

impl ThrottleKey for ByForwardedIp {
    async fn throttle_key(ctx: &RequestContext) -> tork::Result<String> {
        let ip = ctx
            .headers()
            .get("x-forwarded-for")
            .and_then(|value| value.to_str().ok())
            .and_then(|value| value.split(',').next())
            .unwrap_or("unknown")
            .trim();
        Ok(ip.to_string())
    }
}

#[get("/me", throttle(limit = 10, ttl = 60, key = ByForwardedIp))]
pub async fn me() -> tork::Result<&'static str> {
    Ok("me")
}
```

A key works with the inline form and with the parenthesized named form, `throttle(name = "strict", key = ByForwardedIp)`. The bare `throttle = "strict"` shorthand always uses the default IP key.
