# File Uploads and Forms

HTML forms and file uploads use two encodings. A plain key-value form is `application/x-www-form-urlencoded`. A form that carries files is `multipart/form-data`. Tork reads both, with two file types and two handler styles.

## Two file types

A file field binds to one of two types depending on how you handle the file.

`FileBytes` holds the whole file in memory. Use it for small files you read in one go, such as an avatar, a CSV, or a thumbnail.

```rust
file.len();         // usize, size in bytes
file.is_empty();    // bool
file.bytes();       // &[u8]
file.into_bytes();  // Bytes, by value
file.filename();    // Option<&str>, the client-supplied name
file.content_type();// Option<&Mime>, the declared type
```

`UploadFile` stays in memory up to a threshold, then spills to a temporary file on disk. Use it for large uploads you stream to their final destination. It exposes metadata up front and reads its contents through async methods, so disk IO never blocks the runtime.

```rust
file.filename();      // Option<&str>
file.content_type();  // Option<&Mime>
file.size();          // u64, total bytes

let bytes = file.read().await?;        // read it all into Bytes
while let Some(chunk) = file.read_chunk(64 * 1024).await? { /* ... */ }
file.seek_start().await?;              // rewind
file.save_to("/var/uploads/photo.jpg").await?;          // write to a path
let path = file.save_to_dir("/var/uploads", "photo.jpg").await?; // returns the PathBuf
```

`Mime` is re-exported as `tork::mime::Mime`, so you can name it without adding the `mime` crate yourself.

## Two handler styles

Tork offers a parameter style for quick handlers and a model style for forms you want validated and documented. They share the same field types and validation, so pick whichever reads better.

A handler declares exactly one body. Mixing a JSON body (`Valid` or `Json`) with `#[form]` or `#[file]` parameters, or combining `Form` and `Multipart`, is a compile error.

### Parameter style

Annotate each parameter with `#[file]` for a file field or `#[form]` for a text field. The field name defaults to the parameter name. A bare `FileBytes` or `UploadFile` parameter is treated as a file field even without the attribute. Use `#[file(name = "...")]` or `#[form(name = "...")]` when the form field name differs from the parameter.

```rust
use tork::{post, FileBytes};

#[post("/files")]
pub async fn create_file(
    #[file] avatar: FileBytes,
    #[form] caption: String,
) -> tork::Result<FileInfo> {
    Ok(FileInfo { size: avatar.len(), caption })
}
```

Optional and repeated fields use `Option<T>` and `Vec<T>`:

```rust
use tork::{post, UploadFile};

#[post("/docs")]
pub async fn upload(
    #[file] documents: Vec<UploadFile>, // zero or more files under "documents"
    #[form] note: Option<String>,       // an optional text field
) -> tork::Result<()> {
    // ...
    Ok(())
}
```

### Model style

Describe the form once with `#[derive(FormModel)]` and take it as `Multipart<T>`. Each field becomes a form field, validated and documented.

```rust
use tork::{post, FileBytes, FormModel, Multipart, UploadFile};

#[derive(FormModel)]
pub struct ProfileForm {
    #[file(max_size = "2MB", content_types = ["image/png", "image/jpeg"], sniff = true)]
    pub avatar: FileBytes,
    #[file(max_size = "20MB")]
    pub attachment: Option<UploadFile>,
    #[field(min_length = 2, max_length = 40)]
    pub display_name: String,
    pub tags: Vec<String>,
}

#[post("/profile")]
pub async fn save_profile(form: Multipart<ProfileForm>) -> tork::Result<ProfileOut> {
    let form = form.into_inner();
    // form.avatar, form.attachment, form.display_name, form.tags
    Ok(/* ... */)
}
```

## A complete example

This app accepts a profile upload (one image plus text fields) the model way, a quick avatar upload the parameter way, and a plain login form. The form, the routes, the wiring, and a test each live in their own file.

::code-group

```rust [src/forms.rs]
use tork::{api_model, FileBytes, FormModel, UploadFile};

#[derive(FormModel)]
pub struct ProfileForm {
    #[file(max_size = "2MB", content_types = ["image/png", "image/jpeg"], sniff = true)]
    pub avatar: FileBytes,
    #[file(max_size = "20MB")]
    pub attachment: Option<UploadFile>,
    #[field(min_length = 2, max_length = 40)]
    pub display_name: String,
    pub tags: Vec<String>,
}

#[api_model]
pub struct ProfileOut {
    pub display_name: String,
    pub avatar_bytes: usize,
}

#[api_model]
pub struct LoginForm {
    pub username: String,
    pub password: String,
}
```

```rust [src/routers.rs]
use tork::{api_model, api_router, post, FileBytes, Form, Multipart};

use crate::forms::{LoginForm, ProfileForm, ProfileOut};

#[api_model]
pub struct FileInfo {
    pub size: usize,
    pub caption: String,
}

#[api_router(prefix = "/account", tags = ["account"])]
pub mod account_router {
    use super::*;

    // Parameter style: one file and one text field.
    #[post("/avatar")]
    pub async fn upload_avatar(
        #[file] avatar: FileBytes,
        #[form] caption: String,
    ) -> tork::Result<FileInfo> {
        Ok(FileInfo { size: avatar.len(), caption })
    }

    // Model style: the whole form is described once and validated.
    #[post("/profile")]
    pub async fn save_profile(form: Multipart<ProfileForm>) -> tork::Result<ProfileOut> {
        let form = form.into_inner();
        Ok(ProfileOut {
            display_name: form.display_name,
            avatar_bytes: form.avatar.len(),
        })
    }

    // Urlencoded form, no files.
    #[post("/login")]
    pub async fn login(form: Form<LoginForm>) -> tork::Result<&'static str> {
        let _form = form.into_inner();
        Ok("ok")
    }
}

pub use account_router::router;
```

```rust [src/lib.rs]
pub mod forms;
pub mod routers;
```

```rust [src/main.rs]
use tork::{App, UploadConfig};

use myapp::routers;

#[tork::main]
async fn main() -> tork::Result<()> {
    App::new()
        .upload_config(
            UploadConfig::new()
                .max_body_size_mb(32)
                .max_file_size_mb(16)
                .max_files(16),
        )
        .include_router(routers::router())
        .serve("0.0.0.0:8000")
        .await
}
```

```rust [tests/uploads.rs]
use tork::testing::TestClient;
use tork::App;

use myapp::routers;

async fn client() -> tork::Result<TestClient> {
    let app = App::new().include_router(routers::router());
    Ok(TestClient::new(app.build_test().await?).await?)
}

#[tokio::test]
async fn save_profile_accepts_a_png() -> tork::Result<()> {
    let client = client().await?;

    // The 8-byte PNG signature, enough for the sniff check to recognize a PNG.
    let png = [0x89, b'P', b'N', b'G', 0x0D, 0x0A, 0x1A, 0x0A];

    let res = client
        .post("/account/profile")
        .multipart()
        .file_bytes("avatar", "avatar.png", "image/png", png.to_vec())
        .text("display_name", "Ada")
        .text("tags", "rust")
        .send()
        .await?;

    assert_eq!(res.status(), 200);
    Ok(())
}
```

With the app running (`cargo run`), the same upload from the command line:

```bash
curl -F 'avatar=@avatar.png;type=image/png' \
     -F 'display_name=Ada' \
     -F 'tags=rust' \
     http://localhost:8000/account/profile
```
