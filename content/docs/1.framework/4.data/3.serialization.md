# Serialization

A handler returns `tork::Result<T>` and Tork turns the value into an HTTP
response. This page covers how that conversion works and how to control it.

## How responses are built

The generated route code calls one of two internal functions based on whether
you declare a `response_model`:

- `__finish(result, status)` — serializes `T` directly as JSON
- `__finish_into(result, status)` — converts `T` into `U` via `From<T>`, then
  serializes `U`

Both set `Content-Type: application/json` and the status code you specify.

## Default response model

Without any annotation, the return type is the response model.

```rust
use tork::{get, api_model};

#[api_model]
pub struct UserOut {
    pub id: i64,
    pub name: String,
}

#[get("/me")]
pub async fn me() -> tork::Result<UserOut> {
    Ok(UserOut { id: 1, name: "Alice".into() })
}
```

The response is `200` with `{"id":1,"name":"Alice"}`.

## Custom status code

Set a different status with `status_code` on the route attribute.

```rust
use tork::post;

#[post("/users", status_code = 201)]
pub async fn create() -> tork::Result<UserOut> {
    Ok(UserOut { id: 1, name: "Alice".into() })
}
```

## Separate response model

When the return type differs from the API response shape, declare a
`response_model`. The return type must implement `Into<ResponseModel>`.

```rust
use tork::{post, api_model, Valid};

#[api_model]
pub struct CreateUser {
    pub name: String,
}

#[api_model]
pub struct UserOut {
    pub id: i64,
    pub name: String,
}

impl From<CreateUser> for UserOut {
    fn from(input: CreateUser) -> Self {
        UserOut { id: 0, name: input.name }
    }
}

#[post("/users", response_model = UserOut, status_code = 201)]
pub async fn create(body: Valid<CreateUser>) -> tork::Result<CreateUser> {
    Ok(body.into_inner())
}
```

The response uses the `UserOut` schema in OpenAPI. The `From` impl converts the
domain value before serialization.

## IntoResponse trait

Every Tork response path ends up at `IntoResponse`. You can implement it on your
own types for full control.

```rust
use tork::{IntoResponse, Response};

pub struct HtmlPage(pub String);

impl IntoResponse for HtmlPage {
    fn into_response(self) -> Response {
        tork::response::bytes_response(
            tork::StatusCode::OK,
            "text/html; charset=utf-8",
            self.0.into(),
        )
    }
}
```

Built-in implementations:

| Type | Produces |
|---|---|
| `Response` | passed through as is |
| `StatusCode` | empty body with that status |
| `()` | empty body with `200` |
| `Result<T, E>` | forwards Ok, converts Err via `Into<Error>` |
| `Json<T>` | serializes T with `200` and `application/json` |
| `Error` | renders the standard error JSON body |

## Json wrapper

The `Json<T>` wrapper serializes any `Serialize` type. Use it in a handler when
you want to return a value without a response model.

```rust
use serde_json::json;
use tork::{get, Json};

#[get("/health")]
pub async fn health() -> tork::Result<Json<serde_json::Value>> {
    Ok(Json(json!({ "status": "ok" })))
}
```

`Json` is also an extractor for reading JSON bodies without validation.

## Helper functions

For custom responses that do not fit the JSON shape, use the helper functions.

```rust
use bytes::Bytes;
use tork::{StatusCode, response::{bytes_response, json_response}};

// Raw bytes with a content type
bytes_response(StatusCode::OK, "text/plain; charset=utf-8", Bytes::from("hello"));

// JSON with a specific status
json_response(StatusCode::CREATED, &my_value);
```

## OpenAPI schema

The `response_model` and `#[api_model]` types produce the JSON Schema that
appears in the OpenAPI document. Each model field becomes a schema property.

```rust
#[api_model]
pub struct UserOut {
    pub id: i64,

    #[field(title = "Display name", description = "The name shown in the UI")]
    pub display_name: String,
}
```

The generated OpenAPI schema includes the `title` and `description` for that
field. Nested models are registered under `components.schemas` and referenced
by `$ref`.

## Response body types

Internally, a response body is one of two kinds:

- **Buffered** (`RespBody::new(bytes)`) — the whole body is in memory. Most
  responses use this.
- **Streaming** (`RespBody::stream(stream)`) — the body is sent frame by frame.
  SSE endpoints use this.

You rarely need to work with `RespBody` directly unless you are writing
middleware that inspects or replaces the body.

### Real repo example 1: mod.rs
```rust
//! Response types and the [`IntoResponse`] conversion trait.

use bytes::Bytes;
use http::header::{HeaderValue, CONTENT_TYPE};
use http::StatusCode;

use crate::body::RespBody;

pub mod json;

pub use json::{json_response, Json};

/// The concrete HTTP response type used throughout the framework.
pub type Response = http::Response<RespBody>;

/// Converts a value into an HTTP [`Response`].
///
/// Implemented by the response building blocks ([`Json`], [`Error`], status
/// codes) and by [`Result`], so handlers can return high-level values and have
/// them rendered into a proper HTTP response by the generated route glue.
///
/// [`Error`]: crate::Error
/// [`Result`]: core::result::Result
pub trait IntoResponse {
    /// Consumes `self` and produces the HTTP response to send to the client.
    fn into_response(self) -> Response;
}

/// Builds a response with the given status, `Content-Type`, and body.
pub(crate) fn with_body(status: StatusCode, content_type: &'static str, body: Bytes) -> Response {
    let mut response = http::Response::new(RespBody::new(body));
    *response.status_mut() = status;
    response
        .headers_mut()
        .insert(CONTENT_TYPE, HeaderValue::from_static(content_type));
    response
}

/// Builds a response from a pre-serialized body and an explicit content type.
///
/// Useful for serving content that is generated once and cached, such as the
/// OpenAPI document or a static documentation asset.
pub fn bytes_response(status: StatusCode, content_type: &'static str, body: Bytes) -> Response {
    with_body(status, content_type, body)
}

/// Splits a response into its head and fully-buffered body bytes.
///
/// Used by middleware that needs to inspect or rewrite a response body (such as
/// compression). The body is `Full<Bytes>`, so collecting it is immediate.
pub(crate) async fn into_body_bytes(response: Response) -> (http::response::Parts, Bytes) {
    use http_body_util::BodyExt;
    let (parts, body) = response.into_parts();
    let bytes = body
        .collect()
        .await
        .map(|collected| collected.to_bytes())
        .unwrap_or_default();
    (parts, bytes)
}

/// Builds an empty-bodied response carrying only a status code.
pub(crate) fn empty(status: StatusCode) -> Response {
    let mut response = http::Response::new(RespBody::new(Bytes::new()));
    *response.status_mut() = status;
    response
}

impl IntoResponse for Response {
    fn into_response(self) -> Response {
        self
    }
}

impl IntoResponse for StatusCode {
    fn into_response(self) -> Response {
        empty(self)
    }
}

impl IntoResponse for () {
    fn into_response(self) -> Response {
        empty(StatusCode::OK)
    }
}

impl<T, E> IntoResponse for core::result::Result<T, E>
where
    T: IntoResponse,
    E: Into<crate::error::Error>,
{
    fn into_response(self) -> Response {
        match self {
            Ok(value) => value.into_response(),
            Err(error) => error.into().into_response(),
        }
    }
}

/// Renders a handler's result into a response, serializing the success value to
/// JSON with the given status code.
///
/// This is generated-code support, not part of the user-facing API. Handlers
/// return `Result<T>` where `T` is serializable; the route macro feeds that
/// result and the declared success status here.
///
/// A handler error is returned as `Err` rather than rendered here, so that the
/// dispatch boundary can run lifecycle hooks and exception handlers against the
/// error value before it becomes a response.
#[doc(hidden)]
pub fn __finish<T, E>(
    result: core::result::Result<T, E>,
    status: StatusCode,
) -> crate::error::Result<Response>
where
    T: serde::Serialize,
    E: Into<crate::error::Error>,
{
    match result {
        Ok(value) => Ok(json::json_response(status, &value)),
        Err(error) => Err(error.into()),
    }
}

/// Like [`__finish`], but converts the success value into the declared response
/// model `U` (via `From`) before serializing.
///
/// Generated by the route macro when `response_model` differs from the handler's
/// return type. With `response_model` equal to the return type the conversion is
/// the free identity `From<T> for T`.
#[doc(hidden)]
pub fn __finish_into<T, U, E>(
    result: core::result::Result<T, E>,
    status: StatusCode,
) -> crate::error::Result<Response>
where
    U: From<T> + serde::Serialize,
    E: Into<crate::error::Error>,
{
    match result {
        Ok(value) => Ok(json::json_response(status, &U::from(value))),
        Err(error) => Err(error.into()),
    }
}

mod tests {

        let (unit_parts, unit_body) = into_body_bytes(unit).await;
```

### Real repo example 2: json.rs
```rust
//! JSON response support.

use bytes::Bytes;
use http::StatusCode;
use serde::Serialize;

use crate::constants::APPLICATION_JSON;
use crate::error::Error;
use crate::response::{with_body, IntoResponse, Response};

/// A JSON response wrapper.
///
/// Wrapping a serializable value in `Json` renders it as an `application/json`
/// response body with a `200 OK` status. Handlers usually return the bare model
/// and let the generated route glue serialize it; `Json` is available for when a
/// response body needs to be built explicitly.
pub struct Json<T>(pub T);

impl<T: Serialize> IntoResponse for Json<T> {
    fn into_response(self) -> Response {
        json_response(StatusCode::OK, &self.0)
    }
}

/// Serializes `value` to JSON and builds a response with the given status.
///
/// # Errors
///
/// If serialization fails, a redacted `500 Internal Server Error` is returned so
/// that a partially written or malformed body is never sent to the client.
pub fn json_response<T: Serialize + ?Sized>(status: StatusCode, value: &T) -> Response {
    match serde_json::to_vec(value) {
        Ok(buffer) => with_body(status, APPLICATION_JSON, Bytes::from(buffer)),
        Err(error) => Error::internal("response body serialization failed")
            .with_source(error)
            .into_response(),
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    use crate::response::into_body_bytes;
    use serde::Serialize;

    #[derive(Serialize)]
    struct Payload {
        ok: bool,
    }

    struct BrokenSerialize;

    impl Serialize for BrokenSerialize {
        fn serialize<S>(&self, _serializer: S) -> std::result::Result<S::Ok, S::Error>
        where
            S: serde::Serializer,
        {
            Err(serde::ser::Error::custom("boom"))
        }
    }

    #[tokio::test]
    async fn json_response_serializes_payload() {
        let response = json_response(StatusCode::CREATED, &Payload { ok: true });
        let (parts, body) = into_body_bytes(response).await;

        assert_eq!(parts.status, StatusCode::CREATED);
        assert_eq!(parts.headers["content-type"], APPLICATION_JSON);
        assert_eq!(body, Bytes::from_static(br#"{"ok":true}"#));
    }

    #[tokio::test]
    async fn json_response_redacts_serialize_failures() {
        let response = json_response(StatusCode::OK, &BrokenSerialize);
        let (parts, body) = into_body_bytes(response).await;

        assert_eq!(parts.status, StatusCode::INTERNAL_SERVER_ERROR);
        assert_eq!(parts.headers["content-type"], APPLICATION_JSON);
        assert!(String::from_utf8(body.to_vec())
            .unwrap()
            .contains("INTERNAL_SERVER_ERROR"));
    }

    #[tokio::test]
    async fn json_response_does_not_leak_serialize_error_message() {
        let response = json_response(StatusCode::OK, &BrokenSerialize);
        let (_, body) = into_body_bytes(response).await;
        let body_str = String::from_utf8(body.to_vec()).unwrap();
        assert!(
            !body_str.contains("boom"),
            "serialization error message leaked into response body: {body_str}"
        );
    }

    #[tokio::test]
    async fn json_wrapper_serializes_payload_with_default_ok_status() {
        let response = Json(Payload { ok: true }).into_response();
        let (parts, body) = into_body_bytes(response).await;

        assert_eq!(parts.status, StatusCode::OK);
        assert_eq!(parts.headers["content-type"], APPLICATION_JSON);
        assert_eq!(body, Bytes::from_static(br#"{"ok":true}"#));
    }

    #[tokio::test]
    async fn json_wrapper_accepts_dynamic_json_value() {
        let value = serde_json::json!({ "name": "alice", "age": 30 });
        let response = Json(value).into_response();
        let (parts, body) = into_body_bytes(response).await;

        assert_eq!(parts.status, StatusCode::OK);
        assert_eq!(parts.headers["content-type"], APPLICATION_JSON);
        let parsed: serde_json::Value = serde_json::from_slice(&body).unwrap();
        assert_eq!(parsed["name"], "alice");
        assert_eq!(parsed["age"], 30);
    }

    #[tokio::test]
    async fn json_response_preserves_custom_status_code() {
        let response = json_response(StatusCode::ACCEPTED, &Payload { ok: false });
        let (parts, body) = into_body_bytes(response).await;

        assert_eq!(parts.status, StatusCode::ACCEPTED);
        assert_eq!(parts.headers["content-type"], APPLICATION_JSON);
        assert_eq!(body, Bytes::from_static(br#"{"ok":false}"#));
    }
}
```
