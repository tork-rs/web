# Serialization

A route handler returns `tork::Result<T>`. On success, Tork serializes `T` to JSON and sends it with `Content-Type: application/json`. The value's type just needs to be serializable, which every `#[api_model]` is.

```rust
use tork::{get, api_model};

#[api_model]
pub struct UserOut {
    pub id: i64,
    pub name: String,
}

#[get("/me")]
pub async fn me() -> tork::Result<UserOut> {
    Ok(UserOut { id: 1, name: "Alice".into() })
}
```

The response is `200` with body `{"id":1,"name":"Alice"}`.

Handlers declared with the route macros (`#[get]`, `#[post]`, and so on) always serialize their return value to JSON. To send something else, such as raw bytes or HTML, you build a response by hand in middleware or an exception handler, covered at the end of this page.

## Status code

The default success status is `200`. Set another with `status_code` on the route attribute. It takes an integer like `201`, or a `StatusCode` value.

```rust
use tork::post;

#[post("/users", status_code = 201)]
pub async fn create() -> tork::Result<UserOut> {
    Ok(UserOut { id: 1, name: "Alice".into() })
}
```

## Response model

When the value your handler returns differs from the shape you want on the wire, declare a `response_model`. Tork converts the return value into that model with `From`, then serializes the model. So the return type must implement `Into<ResponseModel>`.

```rust
use tork::{post, api_model, Valid};

#[api_model]
pub struct CreateUser {
    pub name: String,
}

#[api_model]
pub struct UserOut {
    pub id: i64,
    pub name: String,
}

impl From<CreateUser> for UserOut {
    fn from(input: CreateUser) -> Self {
        UserOut { id: 0, name: input.name }
    }
}

#[post("/users", response_model = UserOut, status_code = 201)]
pub async fn create(body: Valid<CreateUser>) -> tork::Result<CreateUser> {
    Ok(body.into_inner())
}
```

The `From` impl runs before serialization, and `UserOut` is the schema shown in OpenAPI. A `response_model` can be a generic type, such as `response_model = Vec<UserOut>` for a handler that returns `Vec<UserOut>`.

## Dynamic JSON

For a response with no fixed model, return a `serde_json::Value`. It serializes like any other value.

```rust
use serde_json::json;
use tork::get;

#[get("/health")]
pub async fn health() -> tork::Result<serde_json::Value> {
    Ok(json!({ "status": "ok" }))
}
```

## OpenAPI schema

The response model (or the return type, when no `response_model` is set) produces the JSON Schema in the OpenAPI document. Each model field becomes a schema property, and `title` and `description` from `#[field(...)]` carry into the schema.

```rust
#[api_model]
pub struct UserOut {
    pub id: i64,

    #[field(title = "Display name", description = "The name shown in the UI")]
    pub display_name: String,
}
```

Nested `#[api_model]` types are registered under `components.schemas` and referenced by `$ref`. See [OpenAPI](/docs/framework/fundamentals/openapi) for the full document.
