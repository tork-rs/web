# Caching

Tork ships a small, store-agnostic cache with TTL support. It is a manual key/value API: you decide what to cache and when. The same handler code works whether values are kept in memory or in Redis, so you can start in memory and swap the backend later without changing handlers.

## Enabling the cache

Configure a cache on the app, then inject it where you need it. The default backend is an in-memory store.

```rust
use tork::{App, Cache};

App::new().cache(Cache::in_memory());
```

Caching is opt-in. Without `App::cache(...)`, injecting a `Cache` fails with a clear error.

## Reading and writing

Inject `Cache` by type. Values are serialized to JSON, so any `serde` type can be cached. Keys are `&str`.

```rust
use tork::{get, Cache};

#[get("/greeting")]
pub async fn greeting(cache: Cache) -> tork::Result<String> {
    cache.set("greeting", &"hello").await?;
    let value: Option<String> = cache.get("greeting").await?;
    Ok(value.unwrap_or_default())
}
```

The methods:

| Method | Effect |
| --- | --- |
| `get::<T>(key)` | Returns the value, or `None` if absent or expired |
| `set(key, &value)` | Stores the value with the cache's default TTL |
| `set_ttl(key, &value, ttl)` | Stores the value, expiring it after `ttl` |
| `get_or_set(key, ttl, init)` | Returns the cached value, or computes, stores, and returns it |
| `delete(key)` | Removes one entry |
| `clear()` | Removes every entry |

`get::<T>` needs `T: DeserializeOwned`, and `set` needs `T: Serialize`. `get` returns `Ok(None)` when the key is absent or has expired.
