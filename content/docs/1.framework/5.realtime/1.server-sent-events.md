# Server-Sent Events

Server-Sent Events (SSE) is a one-way streaming protocol over plain HTTP. The server holds the response open and pushes messages as they happen. It fits live feeds, progress updates, notifications, and token-by-token model output. You return an `Sse<T>` built from a stream, and Tork handles the headers, the wire encoding, and the keep-alive.

## A first stream

Declare an endpoint with `#[sse]`. The handler returns `tork::Result<Sse<T>>`, where `T` is the type each event carries. `Sse::new` accepts any stream of `tork::Result<T>`.

```rust
use tork::{sse, Sse};

#[sse("/items/stream", event = "item_update", response_model = ItemOut)]
pub async fn stream_items(service: ItemService) -> tork::Result<Sse<ItemOut>> {
    let items = service.item_stream(); // any Stream<Item = tork::Result<ItemOut>>
    Ok(Sse::new(items))
}
```

The stream can come from anywhere: a channel, a database cursor, or a crate like `async-stream`. Tork does not pick one for you.

The response headers are set automatically:

```
HTTP/1.1 200 OK
Content-Type: text/event-stream
Cache-Control: no-cache
X-Accel-Buffering: no
```

Each item is serialized to JSON and written as one event, named by `event = ...`:

```
event: item_update
data: {"id":1,"name":"Keyboard"}

event: item_update
data: {"id":2,"name":"Mouse"}
```

`#[sse]` handlers take extractors, dependencies, and path parameters exactly like a normal route.
