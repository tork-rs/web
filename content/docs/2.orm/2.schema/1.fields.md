# Fields

A field on a model maps directly to a column. Without any attribute, tork-orm picks the natural SQL type for the Rust type. The `#[field(...)]` attribute lets you override the type, add constraints, mark lifecycle columns, or declare a foreign key.

## Type attributes

### `varchar(length = N)`

Stores the field as a bounded `VARCHAR(N)` instead of unbounded `TEXT`.

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50))]
    pub username: String,
    #[field(varchar(length = 255))]
    pub email: String,
}
```

Use `VARCHAR` when the database should enforce a maximum length. For content fields where length varies widely, plain `String` (TEXT) is simpler.

## Identity and constraint attributes

### `primary_key`

Marks this field as the primary key. Every model needs exactly one.

### `auto`

Makes the primary key auto-incrementing. SQLite renders it as `INTEGER PRIMARY KEY AUTOINCREMENT`, PostgreSQL as `BIGSERIAL`, and MySQL as `BIGINT AUTO_INCREMENT`.

Always set `id: 0` when building a value to insert; the ORM leaves an `auto` column out of the `INSERT`, so the database assigns the real id.

```rust
let user = User::create(
    &db,
    &User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
)
.await?;
println!("{}", user.id); // assigned by the database
```

### `unique`

Adds a `UNIQUE` constraint to the column.

```rust
#[field(varchar(length = 255), unique)]
pub email: String,
```

### `index`

Creates a secondary index on the column (not unique).

```rust
#[field(index)]
pub user_id: i64,
```

For compound indexes or indexes with options (partial, descending, functional), use the `#[table(indexes = [...])]` attribute instead.

A foreign-key column is indexed for you automatically. Pass `index = false` to turn that off:

```rust
#[field(foreign_key = User::id, index = false)]
pub user_id: i64,
```

## Default values

### `default = "expr"`

Sets a SQL-level column default. The value is a string that is embedded verbatim in the `DEFAULT` clause.

```rust
#[field(default = "true")]
pub is_active: bool,

#[field(default = "0")]
pub view_count: i64,
```

### `default_now` and `default_uuid`

Convenient database-side defaults. `default_now` renders `CURRENT_TIMESTAMP` (it needs a timestamp field), and `default_uuid` generates a UUID per dialect (`gen_random_uuid()` on PostgreSQL, `UUID()` on MySQL, a random hex blob on SQLite; it needs a `Uuid` field).

```rust
#[field(default_now)]
pub created_at: OffsetDateTime,

#[field(default_uuid)]
pub public_id: Uuid,
```

### `default_with = "<expr>"`

Runs a Rust expression at insert time to fill the field when it is still at its zero value. This is client-side, before the `INSERT`. The string is a real expression, so it ends in a call.

```rust
fn new_api_token() -> String {
    // generate a token
    "tok_abc".into()
}

#[field(default_with = "new_api_token()")]
pub token: String,
```

The expression runs only when the field is at its type's default (`""` for `String`, `0` for integers, `false` for bool, `None` for `Option<T>`). A field you set explicitly is left alone. `default_with` cannot be combined with `default`, `default_now`, or `default_uuid` on the same field.

### `column = "..."`

Maps the field to a column with a different name. Without it, the column name is the field name.

```rust
#[field(column = "email_address")]
pub email: String,
```

## Lifecycle columns

### `created_at`

Auto-set to the current timestamp at insert time. Never updated after that.

```rust
use time::OffsetDateTime;

#[field(created_at)]
pub created_at: OffsetDateTime,
```

### `updated_at`

Auto-set to the current timestamp at insert time and refreshed on every `model.save(&db)` call.

```rust
#[field(updated_at)]
pub updated_at: OffsetDateTime,
```

### `deleted_at`

Marks this column as the soft-delete timestamp. When present, `model.delete(&db)` stamps the column instead of removing the row, and `Model::query()` automatically filters out rows where `deleted_at IS NOT NULL`.

```rust
#[field(deleted_at)]
pub deleted_at: Option<OffsetDateTime>,
```

Three scope modifiers on `QuerySet` control visibility:

```rust
// Default scope (deleted_at IS NULL)
Note::query().all(&db).await?;

// Include deleted rows
Note::query().with_deleted().all(&db).await?;

// Only deleted rows
Note::query().only_deleted().all(&db).await?;
```

Restore a soft-deleted row with `.restore(&db)`. Remove it permanently with `.force_delete(&db)`.

### `version`

Enables optimistic locking. `model.save(&db)` appends `WHERE version = N` to the update and returns `OrmError::Conflict` when the row was modified by another writer since it was loaded.

```rust
#[field(version)]
pub version: i64,
```

## Foreign keys

### `foreign_key = OtherModel::field`

Declares a foreign key reference, written as `Type::column`. The column keeps the SQL type of its own Rust field, so give it the same type as the column it points at (here both are `i64`).

```rust
#[field(foreign_key = User::id)]
pub user_id: i64,
```

### `on_delete = "..."` and `on_update = "..."`

Sets the referential action. The accepted values are `cascade`, `set_null`, `set_default`, `restrict`, and `no_action`.

```rust
#[field(foreign_key = User::id, on_delete = "cascade")]
pub user_id: i64,
```

Foreign-key columns are indexed automatically so lookups by the key column are efficient (turn that off with `index = false`).

## Database enums

### `db_enum`

Stores a `#[derive(DbEnum)]` enum as text in the database.

```rust
#[derive(Debug, Clone, DbEnum)]
pub enum UserRole {
    Admin,
    Member,
    Guest,
}

#[field(db_enum)]
pub role: UserRole,
```

Variants are stored as their Rust name in `snake_case` by default, and a value that does not match a known variant is an error when the row loads. The storage type follows the dialect: SQLite uses `TEXT` with a `CHECK`, PostgreSQL uses `VARCHAR(255)` with a `CHECK`, and MySQL uses a native `ENUM`. See [Defining models](/docs/orm/getting-started/defining-models) for the `rename_all` and per-variant `rename` options.

## Nullability

Wrap any type in `Option<T>` to allow `NULL`:

```rust
pub bio: Option<String>,          // nullable TEXT
pub last_login_at: Option<OffsetDateTime>, // nullable timestamp
pub parent_id: Option<i64>,       // nullable FK
```

A bare type (`String`, `i64`, `bool`) always produces `NOT NULL`.

## Table-level attributes

Some constraints span multiple columns and live on `#[table]` rather than on a single field.

### Compound indexes

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "posts", indexes = [
    index(fields = [user_id, created_at(desc)]),
    unique(fields = [user_id, slug]),
])]
pub struct Post {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(foreign_key = User::id)]
    pub user_id: i64,
    #[field(varchar(length = 200))]
    pub slug: String,
    pub title: String,
    #[field(created_at)]
    pub created_at: time::OffsetDateTime,
}
```

### Partial indexes

A partial index applies only to rows that satisfy a predicate. Use `Expr::col` from `tork_orm::prelude` to build the condition.

```rust
#[table(name = "jobs", indexes = [
    index(fields = [scheduled_at], where = "status = 'pending'"),
])]
```

### CHECK constraints

```rust
#[table(name = "products", check = "price_cents > 0")]
pub struct Product {
    // ...
    pub price_cents: i64,
}
```

## Complete attribute reference

| Attribute | Placement | Effect |
| --- | --- | --- |
| `primary_key` | `#[field]` | Primary key column |
| `auto` | `#[field]` | Auto-incrementing |
| `unique` | `#[field]` | UNIQUE constraint |
| `index` | `#[field]` | Secondary index |
| `varchar(length = N)` | `#[field]` | VARCHAR(N) instead of TEXT |
| `default = "expr"` | `#[field]` | SQL DEFAULT expression |
| `default_with = "fn"` | `#[field]` | Client-side default function |
| `created_at` | `#[field]` | Auto-stamp on insert |
| `updated_at` | `#[field]` | Auto-stamp on insert and save |
| `deleted_at` | `#[field]` | Soft-delete timestamp + default scope |
| `version` | `#[field]` | Optimistic locking counter |
| `foreign_key = M::f` | `#[field]` | FK reference with matching type |
| `on_delete = "..."` | `#[field]` | Referential delete action |
| `on_update = "..."` | `#[field]` | Referential update action |
| `db_enum` | `#[field]` | Text-backed `#[derive(DbEnum)]` |
| `indexes = [...]` | `#[table]` | Compound or partial indexes |
| `check = "expr"` | `#[table]` | Table-level CHECK constraint |
