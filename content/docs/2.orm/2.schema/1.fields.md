# Fields

Each field on a model maps to a column. The `#[field(...)]` attribute refines how
that column is generated. Omit it and tork-orm picks the natural SQL type for the
Rust type.

```rust
#[derive(Model)]
#[table(name = "posts")]
pub struct Post {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 200))]
    pub title: String,
    #[field(default = "true")]
    pub published: bool,
    pub summary: Option<String>, // nullable
}
```

## Attribute reference

| Attribute | Effect |
| --- | --- |
| `primary_key` | Marks the primary key column. |
| `auto` | Auto-incrementing (BIGSERIAL / IDENTITY). |
| `unique` | Adds a UNIQUE constraint. |
| `varchar(length = N)` | Sized text column instead of TEXT. |
| `default = "…"` | Column default expression. |
| `index` | Creates a secondary index on the column. |

## Nullability

`Option<T>` produces a nullable column; a bare `T` is `NOT NULL`. Changing a field
between the two is a schema change, so it generates a migration like any other —
covered in [migrations](/docs/orm/schema/migrations).
