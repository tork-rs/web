# Relations

Relations connect models through typed foreign keys. Declare them with the
`#[relation]` attribute; tork-orm generates the key column and the join helpers.

```rust
#[derive(Model)]
#[table(name = "posts")]
pub struct Post {
    #[field(primary_key, auto)]
    pub id: i64,
    pub title: String,
    #[relation(belongs_to = User)]
    pub author_id: i64,
}

#[derive(Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[relation(has_many = Post)]
    pub posts: Vec<Post>,
}
```

## Loading related rows

Eager-load a relation in a query so the join happens in one round trip:

```rust
let users = User::query()
    .with(User::posts)
    .all(&db).await?; // each User arrives with its posts loaded
```

A `belongs_to` generates a foreign-key column and a `NOT NULL` constraint unless
the field is `Option`. The reverse `has_many` adds no column — it reads through
the child's foreign key.
