# Relations

Relations connect models through foreign keys. You declare them once in a `#[relations]` block, and the ORM generates typed join helpers and a preload mechanism that batches related rows in a single query.

## Declaring relations

Use the `#[relations]` attribute on an `impl` block. Each method declares one relation:

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50))]
    pub username: String,
    pub is_active: bool,
}

#[derive(Debug, Clone, Model)]
#[table(name = "posts")]
pub struct Post {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(foreign_key = User::id)]
    pub user_id: i64,
    pub title: String,
    pub published: bool,
}

#[relations]
impl User {
    #[has_many(Post, foreign_key = Post::user_id)]
    pub fn posts() {}
}

#[relations]
impl Post {
    #[belongs_to(User, foreign_key = Post::user_id)]
    pub fn author() {}
}
```

`User::posts()` and `Post::author()` now return typed `Relation` handles that can be used with `.join()` and `.preload()`.

## Filtering through a relation with join

`.join(relation)` adds a SQL `INNER JOIN` so you can filter the parent by properties of the child, or vice versa.

```rust
// Users who have at least one published post
let active_authors = User::query()
    .join(User::posts())
    .filter(Post::published.eq(true))
    .all(&db)
    .await?;

// Posts whose author is active
let live_posts = Post::query()
    .join(Post::author())
    .filter(User::is_active.eq(true))
    .all(&db)
    .await?;
```

Note that joining without `distinct` can return duplicate parent rows when the child has multiple matching rows. Add `.distinct()` or adjust the query if you need unique parents.

## Eager loading with preload

`.preload(relation)` loads the related rows in a second query, keyed on the parent ids. This is the standard way to avoid N+1 queries.

```rust
let users = User::query()
    .order_by(User::id.asc())
    .preload(User::posts())
    .all(&db)
    .await?;

// users is Vec<Preloaded<User>>
// Deref to User works for any field on User
for user in &users {
    println!("{}: {} posts", user.username, user.get::<Post>().len());
}
```

The return type is `Vec<Preloaded<User>>` instead of `Vec<User>`. `Preloaded<T>` derefs to `T`, so you read `user.username`, `user.id`, and so on exactly as before. Call `.get::<ChildModel>()` to get the preloaded slice.

## Filtering and ordering preloaded rows

You can constrain which related rows are loaded, and in what order, by chaining methods on the relation before passing it to `.preload()`:

```rust
let users = User::query()
    .filter(User::is_active.eq(true))
    .preload(
        User::posts()
            .filter(Post::published.eq(true))
            .order_by(Post::id.desc())
            .limit(5),
    )
    .all(&db)
    .await?;

for user in &users {
    let posts = user.get::<Post>();
    // At most 5 published posts per user, newest first.
    println!("{}: {} posts", user.username, posts.len());
}
```

The `.filter()`, `.order_by()`, and `.limit()` calls on the relation affect only the preloaded rows. They do not change the parent query.

## belongs_to: loading the parent

`belongs_to` works the same way from the child side:

```rust
let posts = Post::query()
    .preload(Post::author())
    .all(&db)
    .await?;

for post in &posts {
    let authors = post.get::<User>();
    if let Some(author) = authors.first() {
        println!("{} by {}", post.title, author.username);
    }
}
```

## Self-joins

A model can reference itself. Use a type alias or reference the same struct:

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "categories")]
pub struct Category {
    #[field(primary_key, auto)]
    pub id: i64,
    pub name: String,
    #[field(foreign_key = Category::id, on_delete = "set_null")]
    pub parent_id: Option<i64>,
}

#[relations]
impl Category {
    #[has_many(Category, foreign_key = Category::parent_id)]
    pub fn children() {}
}
```

```rust
let top_level = Category::query()
    .filter(Category::parent_id.is_null())
    .preload(Category::children())
    .all(&db)
    .await?;

for cat in &top_level {
    let subs = cat.get::<Category>();
    println!("{} has {} subcategories", cat.name, subs.len());
}
```

## Joining for aggregates

Combine a join with `select` and `group_by` to compute aggregates across related rows. `all_as::<T>` maps each row into a result type that derives `QueryResult` (or `Model`), matching by column name, so the aliases you pick must match the struct field names.

```rust
use tork_orm::prelude::*;

#[derive(Debug, QueryResult)]
pub struct AuthorStats {
    pub user_id: i64,
    pub username: String,
    pub post_count: i64,
}

let stats = User::query()
    .select((
        User::id.as_("user_id"),
        User::username,
        Post::id.count().as_("post_count"),
    ))
    .join(User::posts())
    .group_by((User::id, User::username))
    .order_by(Post::id.count().desc())
    .all_as::<AuthorStats>(&db)
    .await?;

for row in &stats {
    println!("{}: {} posts", row.username, row.post_count);
}
```

## Many-to-many through a join model

tork-orm does not have a special `has_many_through` attribute. Model the join table explicitly and declare two `belongs_to` relations on it:

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "post_tags")]
pub struct PostTag {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(foreign_key = Post::id, on_delete = "cascade")]
    pub post_id: i64,
    #[field(foreign_key = Tag::id, on_delete = "cascade")]
    pub tag_id: i64,
}

#[derive(Debug, Clone, Model)]
#[table(name = "tags")]
pub struct Tag {
    #[field(primary_key, auto)]
    pub id: i64,
    pub name: String,
}

#[relations]
impl PostTag {
    #[belongs_to(Post, foreign_key = PostTag::post_id)]
    pub fn post() {}

    #[belongs_to(Tag, foreign_key = PostTag::tag_id)]
    pub fn tag() {}
}
```

To load all tags for a post, query `PostTag` and preload `.tag()`:

```rust
let tag_rows = PostTag::query()
    .filter(PostTag::post_id.eq(post_id))
    .preload(PostTag::tag())
    .all(&db)
    .await?;

let tags: Vec<&Tag> = tag_rows.iter()
    .flat_map(|pt| pt.get::<Tag>())
    .collect();
```

## Summary

| Method | What it does |
| --- | --- |
| `.join(Relation)` | INNER JOIN for cross-model filtering |
| `.preload(Relation)` | Batch-load related rows, no N+1 |
| `.preload(Rel.filter(...))` | Batch-load with a constraint on the related rows |
| `.preload(Rel.order_by(...))` | Batch-load in a specific order |
| `.preload(Rel.limit(N))` | Cap the related rows per parent |
| `loaded.get::<Child>()` | Access the preloaded slice on a `Preloaded<T>` |
