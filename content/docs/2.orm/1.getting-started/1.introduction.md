# An ORM that compiles to your schema

`tork-orm` is the data layer of the Tork toolchain. You derive `Model` on a
plain Rust struct, and the ORM forges the table, its constraints, and a
reversible migration from that same source of truth — all verified by the
compiler before anything touches a database.

```rust
use tork_orm::Model;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50), unique)]
    pub username: String,
    pub is_active: bool,
}
```

That single struct gives you a typed table definition, a query builder whose
columns are checked at compile time, and a generated migration you can roll
forward and back.

## What you get

- **One source of truth.** The struct is the schema. There is no second YAML or
  SQL file to keep in sync.
- **Reversible migrations.** Every change ships an `up()` and a `down()`, tracked
  in `_tork_migrations`.
- **Typed queries.** A wrong column name or type is a build error, not a runtime
  surprise.
