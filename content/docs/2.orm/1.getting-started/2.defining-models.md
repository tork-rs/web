# Defining models

A model is a plain struct with `#[derive(Model)]`. The `#[table]` attribute names
the table; field attributes describe each column. The derive generates the table
definition, a typed column set for queries, and a migration.

```rust
use tork_orm::Model;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50), unique)]
    pub username: String,
    #[field(varchar(length = 255), unique)]
    pub email: String,
    pub is_active: bool,
}
```

## What it forges

From the struct above, tork-orm derives this table:

| Column | Type | Constraints |
| --- | --- | --- |
| `id` | BIGSERIAL | PRIMARY KEY |
| `username` | VARCHAR(50) | UNIQUE · NOT NULL |
| `email` | VARCHAR(255) | UNIQUE · NOT NULL |
| `is_active` | BOOLEAN | NOT NULL |

A field without an attribute maps to the natural SQL type for its Rust type, and
`Option<T>` becomes a nullable column.
