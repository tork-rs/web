# Defining models

A model is a plain Rust struct with `#[derive(Model)]`. The `#[table]` attribute names the database table. Field attributes describe each column: its type, constraints, and behavior. The derive macro generates the typed column handles, the query builder, and the migration source.

## The minimum viable model

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    pub username: String,
    pub is_active: bool,
}
```

`#[field(primary_key, auto)]` marks the auto-incrementing primary key. Every other `String` field becomes a `TEXT NOT NULL` column. `bool` becomes a boolean. `Option<T>` makes the column nullable.

## Field type mapping

Rust types map to SQL column types automatically. You only need `#[field(...)]` when you want to override the default. The exact SQL type depends on your dialect, chosen when the migration runs.

| Rust type | SQLite | PostgreSQL | MySQL |
| --- | --- | --- | --- |
| `bool` | `BOOLEAN` | `BOOLEAN` | `TINYINT(1)` |
| `i32`, `u32` | `INTEGER` | `INTEGER` | `INT` |
| `i64`, `u64` | `BIGINT` | `BIGINT` | `BIGINT` |
| `f32`, `f64` | `REAL` | `DOUBLE PRECISION` | `DOUBLE` |
| `String` | `TEXT` | `TEXT` | `TEXT` |
| `Vec<u8>` | `BLOB` | `BYTEA` | `BLOB` |
| `OffsetDateTime` | `TIMESTAMP` | `TIMESTAMP WITH TIME ZONE` | `DATETIME` |
| `Uuid` | not supported | `UUID` | `CHAR(36)` |
| `serde_json::Value` | not supported | `JSONB` | `JSON` |
| `Option<T>` | nullable `T` | nullable `T` | nullable `T` |

`OffsetDateTime` is a real timestamp column, not text. SQLite has no native UUID or JSON type, so a SQLite project that uses `Uuid` or `serde_json::Value` is rejected at compile time. For a bounded `VARCHAR`, use `#[field(varchar(length = N))]`, which renders as `VARCHAR(N)` on every dialect.

## Nullable fields

A bare `T` becomes `NOT NULL`. Wrap it in `Option<T>` to allow NULL:

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "posts")]
pub struct Post {
    #[field(primary_key, auto)]
    pub id: i64,
    pub title: String,                   // NOT NULL
    pub body: Option<String>,            // nullable
    pub published: bool,
}
```

## Timestamps

Add `created_at` and `updated_at` timestamps with the corresponding field attributes. The ORM fills them in automatically.

```rust
use time::OffsetDateTime;

#[derive(Debug, Clone, Model)]
#[table(name = "articles")]
pub struct Article {
    #[field(primary_key, auto)]
    pub id: i64,
    pub title: String,
    #[field(created_at)]
    pub created_at: OffsetDateTime,   // set once on insert
    #[field(updated_at)]
    pub updated_at: OffsetDateTime,   // updated on every save
}
```

`created_at` is stamped at insert time and never touched again. `updated_at` is refreshed by every `model.save(&db)` call.

## Soft delete

Add `deleted_at` to make `delete` a soft operation: the row is stamped instead of removed.

```rust
use time::OffsetDateTime;

#[derive(Debug, Clone, Model)]
#[table(name = "notes")]
pub struct Note {
    #[field(primary_key, auto)]
    pub id: i64,
    pub body: String,
    #[field(deleted_at)]
    pub deleted_at: Option<OffsetDateTime>,
}
```

`Note::query()` now filters out deleted rows by default. The QuerySet has three scope modifiers:

```rust
// Default: deleted_at IS NULL (live rows only)
let live = Note::query().all(&db).await?;

// Include every row, deleted or not
let all = Note::query().with_deleted().all(&db).await?;

// Only the soft-deleted rows
let trash = Note::query().only_deleted().all(&db).await?;
```

`note.delete(&db)` stamps `deleted_at`. `note.restore(&db)` clears it. `note.force_delete(&db)` removes the row permanently.

## Optimistic locking

Add a `version` field to prevent lost updates when two writers load the same row concurrently.

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "products")]
pub struct Product {
    #[field(primary_key, auto)]
    pub id: i64,
    pub name: String,
    pub stock: i64,
    #[field(version)]
    pub version: i64,
}
```

`product.save(&db)` appends `WHERE version = <current>` to the update. If the row was modified by someone else since you loaded it, `save` returns an `OrmError::Conflict`. Handle it by reloading and retrying.

```rust
loop {
    let mut p = Product::find(&db, product_id).await?;
    p.stock -= 1;
    match p.save(&db).await {
        Ok(_) => break,
        Err(e) if e.kind() == ErrorKind::Conflict => continue,
        Err(e) => return Err(e),
    }
}
```

## Lifecycle hooks

`ModelHooks` lets you run code before and after each write. The derive generates an empty implementation by default, so to write your own you add the `hooks` flag to `#[table(...)]`. Without it, the manual `impl` would clash with the generated one.

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users", hooks)]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    pub username: String,
    pub email: String,
    pub is_active: bool,
}

impl ModelHooks for User {
    fn before_create(&mut self) {
        self.username = self.username.to_lowercase();
        self.email = self.email.to_lowercase();
    }

    async fn after_create<E: Executor + Send + Sync>(
        &self,
        _executor: &E,
    ) -> tork_orm::Result<()> {
        // Send a welcome email, emit an event, write an audit log...
        Ok(())
    }

    fn before_save(&mut self) {
        self.email = self.email.to_lowercase();
    }
}
```

The trait has six methods, all with a default no-op body, so you override only the ones you need:

| Hook | When | Form |
| --- | --- | --- |
| `before_create(&mut self)` | before an insert | sync |
| `after_create(&self, executor)` | after an insert, on the stored row | async |
| `before_save(&mut self)` | before an update | sync |
| `after_save(&self, executor)` | after an update | async |
| `before_delete(&self)` | before a delete | sync |
| `after_delete(&self, executor)` | after a delete | async |

The `after_*` hooks are async and receive the executor, so they can run more queries. The order for a `create` is:

1. fill any `default_with` fields that are still unset
2. `before_create`
3. the `INSERT` runs
4. `after_create`, which receives the stored row with its database-assigned id

A `save` runs `before_save`, then the `UPDATE`, then `after_save`. A `delete` runs `before_delete`, then the delete, then `after_delete`.

Hooks fire on the per-row methods `create`, `save`, `delete`, `force_delete`, and `upsert` (which runs the create hooks). They are skipped by the bulk paths, `bulk_create` and the `QuerySet` operations like `QuerySet::update`, so those stay fast.

## Database enums

Use `#[derive(DbEnum)]` to store an enum as text in the database. The ORM serializes and deserializes the variants automatically.

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, DbEnum)]
pub enum OrderStatus {
    Pending,
    Paid,
    Shipped,
    Cancelled,
}

#[derive(Debug, Clone, Model)]
#[table(name = "orders")]
pub struct Order {
    #[field(primary_key, auto)]
    pub id: i64,
    pub total_cents: i64,
    #[field(db_enum)]
    pub status: OrderStatus,
}
```

The `#[field(db_enum)]` attribute tells the macro to use the text codec. Each variant is stored as its name in `snake_case` by default (`Pending` becomes `pending`). Change the convention for the whole enum with `#[db_enum(rename_all = "...")]`, or override a single variant with `#[db_enum(rename = "...")]`.

```rust
#[derive(Debug, Clone, DbEnum)]
#[db_enum(rename_all = "SCREAMING_SNAKE_CASE")]
pub enum OrderStatus {
    Pending,                       // stored as "PENDING"
    #[db_enum(rename = "PAID_IN_FULL")]
    Paid,                          // stored as "PAID_IN_FULL"
    Shipped,
    Cancelled,
}
```

`rename_all` accepts `snake_case`, `SCREAMING_SNAKE_CASE`, `kebab-case`, `lowercase`, `UPPERCASE`, `PascalCase`, and `camelCase`. Only unit variants (no data) are supported.

## Client-side defaults

`#[field(default_with = "<expr>")]` runs a Rust expression at insert time to fill a field that is still at its zero value. The string is a real expression, so it ends in a call, like `"new_token()"` or `"Uuid::new_v4()"`. This is useful for tokens, UUIDs, or generated slugs.

```rust
fn new_token() -> String {
    // generate a random token
    "tok_placeholder".into()
}

#[derive(Debug, Clone, Model)]
#[table(name = "api_keys")]
pub struct ApiKey {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(default_with = "new_token()")]
    pub token: String,
    pub user_id: i64,
}
```

`ApiKey::create(&db, &ApiKey { id: 0, token: String::new(), user_id: 42 })` fills `token` from `new_token()` before the insert, because `token` is still the empty string. A field you set explicitly is left alone. For an `Option<T>` field, the expression runs only when the value is `None`.

## The full picture

A real model combining all of the above:

```rust
use time::OffsetDateTime;
use tork_orm::prelude::*;

#[derive(Debug, Clone, DbEnum)]
pub enum UserRole { Admin, Member, Guest }

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,

    #[field(varchar(length = 50), unique)]
    pub username: String,

    #[field(varchar(length = 255), unique)]
    pub email: String,

    #[field(db_enum)]
    pub role: UserRole,

    pub is_active: bool,

    pub last_login_at: Option<OffsetDateTime>,

    #[field(created_at)]
    pub created_at: OffsetDateTime,

    #[field(updated_at)]
    pub updated_at: OffsetDateTime,

    #[field(deleted_at)]
    pub deleted_at: Option<OffsetDateTime>,
}
```
