# Mutations

tork-orm provides typed methods for every kind of write operation. All of them accept `impl Executor`, so the same call works against a pool connection or a transaction handle.

The models used in the examples below:

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50), unique)]
    pub username: String,
    #[field(varchar(length = 255), unique)]
    pub email: String,
    pub is_active: bool,
}
```

## Inserting

### `Model::create(executor, &value)`

Inserts one row and returns the stored model with all database-assigned values (the auto-id, defaults, timestamps) filled in.

```rust
let user = User::create(
    &db,
    &User {
        id: 0,           // ignored; the database assigns the real id
        username: "alice".into(),
        email: "alice@example.com".into(),
        is_active: true,
    },
)
.await?;

println!("created id={}", user.id); // real id from the database
```

`create` fires the `before_create` hook before the insert and `after_create` after.

### `Model::bulk_create(executor, &[values])`

Inserts multiple rows in one or more batched `INSERT` statements. Returns the number of rows inserted.

```rust
let users = vec![
    User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
    User { id: 0, username: "bob".into(),   email: "bob@example.com".into(),   is_active: false },
    User { id: 0, username: "carol".into(), email: "carol@example.com".into(), is_active: true },
];

let count = User::bulk_create(&db, &users).await?;
println!("inserted {count} users");
```

`bulk_create` splits the rows into chunks automatically to stay within the database's bind-parameter limit (999 on SQLite, higher on PostgreSQL and MySQL). Lifecycle hooks are not fired for bulk inserts.

Passing an empty slice is a no-op and returns 0.

## Updating

### `model.save(executor)`

Writes back the current field values to the row identified by the model's primary key. Returns the number of rows affected (0 if no row with that key exists, 1 otherwise).

```rust
let mut user = User::find(&db, 42).await?;
user.email = "alice-new@example.com".into();
user.is_active = false;

let changed = user.save(&db).await?;
assert_eq!(changed, 1);
```

`save` fires `before_save` and `after_save`. If the model has an `updated_at` field, it is refreshed automatically.

### `QuerySet::update(executor, [assignments])`

Updates specific columns on all rows matching the query. Returns the number of rows changed.

```rust
// Deactivate all users except bob
let changed = User::query()
    .filter(User::username.ne("bob"))
    .update(&db, [User::is_active.set(false)])
    .await?;
```

You can set multiple columns at once:

```rust
User::query()
    .filter(User::id.eq(user_id))
    .update(&db, [
        User::email.set("new@example.com"),
        User::is_active.set(true),
    ])
    .await?;
```

You can also set a column to an expression, not just a literal. This is useful for atomic increments:

```rust
#[derive(Debug, Clone, Model)]
#[table(name = "counters")]
pub struct Counter {
    #[field(primary_key, auto)]
    pub id: i64,
    pub hits: i64,
}

// Atomic increment, no read-modify-write race
Counter::query()
    .filter(Counter::id.eq(counter_id))
    .update(&db, [Counter::hits.set(Counter::hits.add(1_i64))])
    .await?;
```

Lifecycle hooks are not fired for `QuerySet::update`. If you need hooks to run, load the model and call `save`.

### `QuerySet::update_returning(executor, [assignments])`

Same as `.update()` but returns the updated rows as `Vec<Model>`, using a SQL `RETURNING` clause. This works on PostgreSQL and SQLite 3.35 or newer. MySQL has no `RETURNING`, so the statement fails at the database there; the ORM does not pre-check the dialect.

```rust
let updated = User::query()
    .filter(User::username.eq("alice"))
    .update_returning(&db, [User::is_active.set(false)])
    .await?;

for user in &updated {
    println!("updated: {} is_active={}", user.username, user.is_active);
}
```

## Deleting

### `model.delete(executor)`

Deletes the row by primary key. Returns the number of rows removed (0 if already gone).

For models with a `deleted_at` column, this stamps the column instead of removing the row (soft delete). Use `model.force_delete()` to remove permanently.

```rust
let user = User::find(&db, 42).await?;
let removed = user.delete(&db).await?;
println!("removed {removed} row(s)");
```

### `QuerySet::delete(executor)`

Deletes all rows matching the query. Returns the row count.

```rust
// Delete a single row by id
let removed = User::query()
    .filter(User::id.eq(42))
    .delete(&db)
    .await?;

// Delete all inactive users
let removed = User::query()
    .filter(User::is_active.eq(false))
    .delete(&db)
    .await?;

// Truncate the table (no filter)
User::query().delete(&db).await?;
```

### `QuerySet::delete_returning(executor)`

Returns the deleted rows instead of a count. Supported on PostgreSQL and SQLite 3.35+.

```rust
let removed = User::query()
    .filter(User::username.eq("alice"))
    .delete_returning(&db)
    .await?;

for user in &removed {
    println!("deleted: {}", user.username);
}
```

## Soft delete

When a model has `#[field(deleted_at)]`, the `delete` methods stamp the field instead of removing the row, and `Model::query()` automatically excludes soft-deleted rows.

```rust
use time::OffsetDateTime;

#[derive(Debug, Clone, Model)]
#[table(name = "notes")]
pub struct Note {
    #[field(primary_key, auto)]
    pub id: i64,
    pub body: String,
    #[field(deleted_at)]
    pub deleted_at: Option<OffsetDateTime>,
}
```

```rust
let note = Note::create(&db, &Note { id: 0, body: "hello".into(), deleted_at: None }).await?;

// Soft-deletes the row (stamps deleted_at)
note.delete(&db).await?;

// Default scope hides it
let live = Note::query().count(&db).await?;     // 0
let all  = Note::query().with_deleted().count(&db).await?;  // 1
let dead = Note::query().only_deleted().count(&db).await?;  // 1

// Restore: clears deleted_at
let restored = Note::query()
    .only_deleted()
    .filter(Note::id.eq(note.id))
    .restore(&db)
    .await?;

// Permanently remove (bypasses soft-delete)
note.force_delete(&db).await?;
Note::query().with_deleted().hard_delete(&db).await?;
```

Soft-delete methods summary:

| Method | Effect |
| --- | --- |
| `note.delete(&db)` | Stamp `deleted_at` |
| `note.force_delete(&db)` | Remove permanently |
| `Note::query().delete(&db)` | Stamp matching rows |
| `Note::query().hard_delete(&db)` | Remove matching rows permanently |
| `Note::query().restore(&db)` | Clear `deleted_at` on matching rows |
| `.with_deleted()` | Include soft-deleted rows |
| `.only_deleted()` | Only soft-deleted rows |

## Upsert

### `Model::upsert(executor, &value)`

Inserts the row if no conflict occurs, or updates it in place if it already exists. Conflicts are detected using the model's primary key.

```rust
let stored = User::upsert(
    &db,
    &User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
)
.await?;
```

### `Model::upsert_on(executor, &value, conflict_targets)`

Upsert on specific unique columns. The database performs `ON CONFLICT (cols) DO UPDATE`.

```rust
// If a user with this username already exists, update their email and status.
let stored = User::upsert_on(
    &db,
    &User { id: 0, username: "alice".into(), email: "alice-v2@example.com".into(), is_active: false },
    &["username"],
)
.await?;

// The returned row has the id of the existing row if one was found.
println!("id={} email={}", stored.id, stored.email);
```

## Find-or-create helpers

### `Model::get_or_create(executor, filter_fn, &value)`

Finds a single row matching the filter. If no row exists, inserts `value`. Returns `(row, created)` where `created` is `true` when a new row was inserted.

Errors with `ErrorKind::MultipleFound` if the filter matches more than one row.

```rust
let (user, created) = User::get_or_create(
    &db,
    |q| q.filter(User::username.eq("alice")),
    &User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
)
.await?;

if created {
    println!("new user created");
} else {
    println!("existing user found: id={}", user.id);
}
```

### `Model::update_or_create(executor, filter_fn, &value)`

Finds a single row matching the filter. If it exists, updates it with the new values. If it does not, inserts it. Returns `(row, created)`.

```rust
let (user, created) = User::update_or_create(
    &db,
    |q| q.filter(User::username.eq("alice")),
    &User { id: 0, username: "alice".into(), email: "alice-updated@example.com".into(), is_active: true },
)
.await?;
```

### `Model::first_or_create(executor, filter_fn, &value)`

Finds the first row matching the filter. If the filter matches nothing, inserts `value`. Unlike `get_or_create`, does not error when the filter matches multiple rows; it returns the first one.

```rust
let user = User::first_or_create(
    &db,
    |q| q.filter(User::username.eq("alice")),
    &User { id: 0, username: "alice".into(), email: "alice@example.com".into(), is_active: true },
)
.await?;
```

## Using mutations inside transactions

All mutation methods accept `impl Executor`. Pass a `&Transaction` handle instead of `&db` and every write participates in the same transaction.

```rust
use tork_orm::transaction;

let (user, post) = transaction!(db, |tx| async move {
    let user = User::create(tx, &User {
        id: 0,
        username: "alice".into(),
        email: "alice@example.com".into(),
        is_active: true,
    })
    .await?;

    let post = Post::create(tx, &Post {
        id: 0,
        user_id: user.id,
        title: "Hello world".into(),
        published: true,
    })
    .await?;

    Ok((user, post))
})
.await?;
```

If either insert fails, both are rolled back. See [Transactions](/docs/orm/querying/transactions) for the full API.

## Quick reference

| Method | Description |
| --- | --- |
| `Model::create(&db, &value)` | Insert one row, return the stored model |
| `Model::bulk_create(&db, &[values])` | Insert many rows |
| `model.save(&db)` | Write all fields back to the matching row |
| `QuerySet::update(&db, [assignments])` | Update columns on matching rows |
| `QuerySet::update_returning(&db, [assignments])` | Same, return the updated rows |
| `model.delete(&db)` | Delete or soft-delete by primary key |
| `model.force_delete(&db)` | Delete permanently (bypass soft-delete) |
| `QuerySet::delete(&db)` | Delete matching rows |
| `QuerySet::delete_returning(&db)` | Delete and return the removed rows |
| `QuerySet::restore(&db)` | Clear `deleted_at` on matching rows |
| `QuerySet::hard_delete(&db)` | Permanently delete soft-deleted rows |
| `Model::upsert(&db, &value)` | Insert or update on primary key |
| `Model::upsert_on(&db, &value, cols)` | Insert or update on specific unique columns |
| `Model::get_or_create(&db, filter, &value)` | Find exactly one or create |
| `Model::update_or_create(&db, filter, &value)` | Find and update, or create |
| `Model::first_or_create(&db, filter, &value)` | Find first or create |

Continue to [Transactions](/docs/orm/querying/transactions) to group mutations atomically.
