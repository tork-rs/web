# Mutations

Writes go through typed model values. Build a row, insert it, and the returned
value carries the database-assigned keys.

```rust
let user = User {
    id: 0, // assigned on insert
    username: "ada".into(),
    email: "ada@example.com".into(),
    is_active: true,
};
let saved = user.insert(&db).await?; // saved.id is now set
```

## Updating

Mutate a loaded model and persist it, or run a targeted update on a filter:

```rust
let mut user = User::query().filter(User::id.eq(1)).one(&db).await?;
user.is_active = false;
user.save(&db).await?;

// or, without loading first:
User::query()
    .filter(User::is_active.eq(false))
    .update(User::is_active.set(true))
    .exec(&db).await?;
```

## Deleting

```rust
User::query()
    .filter(User::id.eq(1))
    .delete()
    .exec(&db).await?;
```

Every mutation is checked against the model, so you can never set a column that
doesn't exist or assign the wrong type.
