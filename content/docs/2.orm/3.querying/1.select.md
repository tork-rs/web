# Selecting rows

`Model::query()` returns a typed builder. Filters, ordering, and projections all
reference the model's columns, so a wrong name or type fails to compile.

```rust
let active = User::query()
    .filter(User::is_active.eq(true))
    .order_by(User::username)
    .all(&db).await?; // Vec<User>
```

## Terminators

| Method | Returns |
| --- | --- |
| `.all(&db)` | `Vec<Model>` |
| `.first(&db)` | `Option<Model>` |
| `.one(&db)` | `Model` (errors if not exactly one) |
| `.count(&db)` | `i64` |

## Composing filters

Filters chain with `and` / `or`, and the column expressions are type-checked
against the field:

```rust
let recent = Post::query()
    .filter(Post::published.eq(true).and(Post::id.gt(1000)))
    .limit(20)
    .all(&db).await?;
```

To change data instead of reading it, see [mutations](/docs/orm/querying/mutations).
