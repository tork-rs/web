# Selecting rows

`Model::query()` returns a `QuerySet<Model>`: a typed, lazy builder. Nothing hits the database until you call a terminal method like `.all()`, `.first()`, or `.count()`. Chain as many builder methods as you like before that.

```rust
use tork_orm::prelude::*;

#[derive(Debug, Clone, Model)]
#[table(name = "users")]
pub struct User {
    #[field(primary_key, auto)]
    pub id: i64,
    #[field(varchar(length = 50))]
    pub username: String,
    pub email: String,
    pub is_active: bool,
}
```

## Terminal methods

These execute the query and return results.

### `.all(executor)`

Returns every matching row as `Vec<Model>`.

```rust
let users = User::query().all(&db).await?;
```

### `.first(executor)`

Returns the first matching row as `Option<Model>`. Returns `None` when no rows match.

```rust
let user = User::query()
    .filter(User::username.eq("alice"))
    .first(&db)
    .await?;

if let Some(u) = user {
    println!("found: {}", u.username);
}
```

### `.one(executor)`

Returns exactly one row. Errors with `ErrorKind::NotFound` when zero rows match, and `ErrorKind::MultipleFound` when more than one row matches.

```rust
let user = User::query()
    .filter(User::username.eq("alice"))
    .one(&db)
    .await?; // errors if alice does not exist or is not unique
```

### `.count(executor)`

Returns the number of matching rows as `i64`.

```rust
let total = User::query().count(&db).await?;

let active_count = User::query()
    .filter(User::is_active.eq(true))
    .count(&db)
    .await?;
```

### `.exists(executor)`

Returns `true` if at least one row matches. Generates `SELECT EXISTS(SELECT 1 FROM ... WHERE ...)` and is more efficient than `count > 0` for pure presence checks.

```rust
let has_admins = User::query()
    .filter(User::is_active.eq(true))
    .exists(&db)
    .await?;
```

### `.chunk(executor, size)`

Splits the result into fixed-size batches. Useful for processing large result sets without loading everything into memory at once. Returns `Vec<Vec<Model>>`.

```rust
let batches = User::query()
    .order_by(User::id.asc())
    .chunk(&db, 100)
    .await?;

for batch in batches {
    for user in batch {
        // process user
    }
}
```

### `.all_as::<T>(executor)`

Fetches rows and maps them into a different type `T`. Useful for custom projections and aggregates. `T` must implement `FromRow`, which `#[derive(QueryResult)]` provides; it maps each row by column name, so your struct fields must match the selected column names (and aliases).

```rust
#[derive(Debug, QueryResult)]
struct UserSummary {
    username: String,
    post_count: i64,
}

let summaries = User::query()
    .select((User::username, Post::id.count().as_("post_count")))
    .join(User::posts())
    .group_by((User::username,))
    .all_as::<UserSummary>(&db)
    .await?;
```

### `Model::find(executor, pk)`

Fetches a single row by primary key. Errors with `ErrorKind::NotFound` when no row with that key exists.

```rust
let user = User::find(&db, 42).await?;
```

### `.none()`

A builder (not a terminal) that forces the query to match nothing, no matter what other filters are set. It adds the predicate `0 = 1`. Useful when you need a guaranteed empty result in conditional code.

```rust
let empty = User::query().none().all(&db).await?;
assert!(empty.is_empty());
```

### `.paginate(executor, page, page_size)`

Fetches one page and the total count in a single call, returning a `Page<Model>` with `items`, `total`, `page`, `page_size`, and `pages`. Pages start at 1.

```rust
let page = User::query()
    .order_by(User::id.asc())
    .paginate(&db, 1, 20)
    .await?;

println!("{} of {} users, {} pages", page.items.len(), page.total, page.pages);
```

Use `.paginate_as::<T>(executor, page, page_size)` to paginate a custom projection.

## Filtering

### `.filter(expr)`: AND predicate

Each `.filter()` call ANDs a predicate to the query.

```rust
let users = User::query()
    .filter(User::is_active.eq(true))
    .filter(User::username.ne("bot"))
    .all(&db)
    .await?;
```

### `.filter_any([exprs])`: OR group

`.filter_any` wraps a list of expressions in an OR clause. Combined with `.filter()`, the result is `AND (A OR B OR ...)`.

```rust
// is_active = true AND (username = 'alice' OR username = 'bob')
let users = User::query()
    .filter(User::is_active.eq(true))
    .filter_any([
        User::username.eq("alice"),
        User::username.eq("bob"),
    ])
    .all(&db)
    .await?;
```

### `.filter_not(expr)`: NOT predicate

Negates the expression.

```rust
// Inactive users only
let inactive = User::query()
    .filter_not(User::is_active.eq(true))
    .all(&db)
    .await?;
```

### `.filter_raw(sql, params)`: raw SQL WHERE

Appends a raw SQL fragment to the WHERE clause, with a `?` placeholder for each bound value. This is for expressions the typed builder cannot generate.

```rust
let users = User::query()
    .filter_raw("LENGTH(username) > ?", [4_i64])
    .all(&db)
    .await?;

// No params version
let users = User::query()
    .filter_raw("username NOT LIKE 'bot_%'", [] as [i64; 0])
    .all(&db)
    .await?;
```

Always use bind parameters. Never interpolate user input into the SQL string.

## Column expressions

Column handles produce typed expressions for filter and order clauses.

### Comparison operators

```rust
User::id.eq(42)            // id = 42
User::id.ne(0)             // id <> 0
User::id.gt(10)            // id > 10
User::id.ge(10)            // id >= 10
User::id.lt(100)           // id < 100
User::id.le(100)           // id <= 100
User::id.between(1, 10)    // id BETWEEN 1 AND 10
```

### NULL checks

```rust
User::bio.is_null()        // bio IS NULL
User::bio.is_not_null()    // bio IS NOT NULL
```

### String matching

```rust
User::username.like("ali%")        // LIKE 'ali%'
User::username.ilike("ALICE")      // ILIKE 'ALICE' (case-insensitive)
User::username.starts_with("ali")  // LIKE 'ali%'
User::username.ends_with("ce")     // LIKE '%ce'
User::username.contains("li")      // LIKE '%li%'
User::username.istarts_with("ALI") // ILIKE 'ALI%'
User::username.iends_with("CE")    // ILIKE '%CE'
User::username.icontains("LI")     // ILIKE '%LI%'
```

### List membership

```rust
User::id.in_list([1, 2, 3])        // id IN (1, 2, 3)
User::id.not_in([4, 5, 6])         // id NOT IN (4, 5, 6)

// Passing an empty list to not_in matches all rows (vacuously true)
// Passing an empty list to in_list matches no rows
```

### Subquery membership

```rust
// Users whose id appears in the active-user subquery
User::query()
    .filter(User::id.in_subquery(
        User::query()
            .filter(User::is_active.eq(true))
            .select((User::id,)),
    ))
    .all(&db)
    .await?;

User::query()
    .filter(User::id.not_in_subquery(
        User::query()
            .filter(User::is_active.eq(false))
            .select((User::id,)),
    ))
    .all(&db)
    .await?;
```

### EXISTS / NOT EXISTS

```rust
// Rows where an active user exists in the table
User::query()
    .filter(Expr::exists(
        User::query().filter(User::is_active.eq(true)),
    ))
    .all(&db)
    .await?;

User::query()
    .filter(Expr::not_exists(
        User::query().filter(User::is_active.eq(false)),
    ))
    .all(&db)
    .await?;
```

### Arithmetic on columns

```rust
// SET hits = hits + 5
Counter::query()
    .update(&db, [Counter::hits.set(Counter::hits.add(5_i64))])
    .await?;

// Column arithmetic in expressions
let expr = User::id.mul(2_i64).add(Expr::value(Value::Int(1)));
```

### CASE expressions

```rust
use tork_orm::prelude::*;

let label = Expr::case()
    .when(User::is_active.eq(true), Expr::value(Value::Text("active".into())))
    .else_(Expr::value(Value::Text("inactive".into())))
    .end()
    .as_("status");
```

Each `.when(condition, result)` adds a branch, `.else_(default)` is optional, and `.end()` finishes the expression. Call `.as_("...")` on the result to label the column.

## Ordering

### `.order_by(term)`

```rust
User::query().order_by(User::username.asc()).all(&db).await?;
User::query().order_by(User::id.desc()).all(&db).await?;
```

Chain multiple columns:

```rust
User::query()
    .order_by(User::is_active.desc())
    .order_by(User::username.asc())
    .all(&db)
    .await?;
```

NULL ordering:

```rust
User::query()
    .order_by(User::last_login_at.desc().nulls_last())
    .all(&db)
    .await?;

User::query()
    .order_by(User::priority.asc().nulls_first())
    .all(&db)
    .await?;
```

## Pagination

### `.limit(n)` and `.offset(n)`

```rust
// Page 1: first 20 rows
let page1 = User::query()
    .order_by(User::id.asc())
    .limit(20)
    .all(&db)
    .await?;

// Page 2: skip 20, take next 20
let page2 = User::query()
    .order_by(User::id.asc())
    .limit(20)
    .offset(20)
    .all(&db)
    .await?;
```

## Joins

`.join(relation)` adds a SQL INNER JOIN. Used with relations declared via `#[relations]`:

```rust
// Posts from active users only
let posts = Post::query()
    .join(Post::author())
    .filter(User::is_active.eq(true))
    .all(&db)
    .await?;

// Users who have at least one published post
let authors = User::query()
    .join(User::posts())
    .filter(Post::published.eq(true))
    .all(&db)
    .await?;
```

When a join can produce duplicates (one user with many posts), add `.distinct()` to deduplicate parent rows:

```rust
let authors = User::query()
    .join(User::posts())
    .filter(Post::published.eq(true))
    .distinct()
    .all(&db)
    .await?;
```

`.left_join(relation)` does a `LEFT JOIN` instead, keeping parent rows that have no match. There are also `.right_join`, `.full_join`, and `.cross_join` for the other join kinds.

## Eager loading with preload

`.preload(relation)` runs one extra query to load related rows, keyed on parent ids. The result is `Vec<Preloaded<Model>>`.

```rust
let users = User::query()
    .preload(User::posts())
    .all(&db)
    .await?;

for user in &users {
    // Deref to the parent
    println!("{}", user.username);
    // Get related rows
    let posts = user.get::<Post>();
    println!("  {} posts", posts.len());
}
```

Filter or order the preloaded rows without affecting the parent query:

```rust
let users = User::query()
    .filter(User::is_active.eq(true))
    .preload(
        User::posts()
            .filter(Post::published.eq(true))
            .order_by(Post::id.desc())
            .limit(3),
    )
    .all(&db)
    .await?;
```

Preloading runs exactly two queries no matter how many parents exist. It is never N+1.

## Custom projections

`.select((col1, col2, ...))` lets you pick specific columns or use aggregate expressions. Follow it with `.all_as::<T>()` to deserialize into a matching struct.

```rust
#[derive(Debug, QueryResult)]
struct AuthorStats {
    user_id: i64,
    username: String,
    post_count: i64,
    total_views: i64,
}

let stats = User::query()
    .select((
        User::id.as_("user_id"),
        User::username,
        Post::id.count().as_("post_count"),
        Post::view_count.sum().as_("total_views"),
    ))
    .join(User::posts())
    .group_by((User::id, User::username))
    .order_by(Post::view_count.sum().desc())
    .all_as::<AuthorStats>(&db)
    .await?;

for row in &stats {
    println!("{}: {} posts, {} views", row.username, row.post_count, row.total_views);
}
```

## Aggregates

Column handles expose aggregate methods. These are usually used with `.select()` and `.group_by()`.

| Expression | SQL |
| --- | --- |
| `Col::field.count()` | `COUNT("field")` |
| `Col::field.sum()` | `SUM("field")` |
| `Col::field.avg()` | `AVG("field")` |
| `Col::field.min()` | `MIN("field")` |
| `Col::field.max()` | `MAX("field")` |

```rust
let post_count_per_user = User::query()
    .select((User::id, User::username, Post::id.count().as_("post_count")))
    .join(User::posts())
    .group_by((User::id, User::username))
    .all_as::<PostCount>(&db)
    .await?;
```

## Locking

### `.for_update()`

Adds `FOR UPDATE` to the query, locking the selected rows until the transaction commits. Used to prevent concurrent writes to the same rows.

```rust
let user = User::query()
    .filter(User::id.eq(user_id))
    .for_update()
    .one(&db)
    .await?;
// Within a transaction, no other transaction can update `user` until this one commits.
```

`for_update` is meaningful only inside a transaction. Chain a modifier for finer control: `.skip_locked()` skips rows another transaction already locked, and `.nowait()` errors instead of waiting. `.for_share()` takes a weaker shared lock. The modifiers require PostgreSQL or MySQL.

## Using a transaction as the executor

Every terminal method accepts `impl Executor`, which both `Database` and `Transaction` implement. You can pass the transaction handle directly:

```rust
db.transaction(|tx| Box::pin(async move {
    let user = User::query()
        .filter(User::id.eq(1))
        .one(tx)
        .await?;

    User::query()
        .filter(User::id.eq(1))
        .update(tx, [User::is_active.set(false)])
        .await?;

    Ok(())
}))
.await?;
```

## Quick reference

| Method | Returns | Description |
| --- | --- | --- |
| `.all(db)` | `Vec<Model>` | All matching rows |
| `.first(db)` | `Option<Model>` | First match or None |
| `.one(db)` | `Model` | Exactly one match, error otherwise |
| `.count(db)` | `i64` | Count of matching rows |
| `.exists(db)` | `bool` | True if any row matches |
| `.chunk(db, n)` | `Vec<Vec<Model>>` | Batched results |
| `.all_as::<T>(db)` | `Vec<T>` | Custom projection (`T: QueryResult`) |
| `.paginate(db, page, size)` | `Page<Model>` | One page plus total count |
| `Model::find(db, pk)` | `Model` | By primary key |
| `.none()` | `QuerySet` | Force empty result |
| `.filter(expr)` | `QuerySet` | AND predicate |
| `.filter_any([exprs])` | `QuerySet` | OR group |
| `.filter_not(expr)` | `QuerySet` | NOT predicate |
| `.filter_raw(sql, [params])` | `QuerySet` | Raw SQL WHERE |
| `.order_by(col.asc())` | `QuerySet` | Order ascending |
| `.order_by(col.desc())` | `QuerySet` | Order descending |
| `.limit(n)` / `.offset(n)` | `QuerySet` | Max rows / skip rows |
| `.join(relation)` / `.left_join(relation)` | `QuerySet` | INNER / LEFT JOIN |
| `.preload(relation)` | `Preloader` | Eager load related rows |
| `.select((cols...))` | `QuerySet` | Custom columns |
| `.group_by((cols...))` / `.having(expr)` | `QuerySet` | GROUP BY / HAVING |
| `.for_update()` | `QuerySet` | Row-level lock |
| `.distinct()` | `QuerySet` | Deduplicate |
